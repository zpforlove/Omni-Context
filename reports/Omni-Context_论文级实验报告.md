# 面向复杂声学场景的"线索式上下文"（AGSC）：让 Omni 模型真实听懂多说话人重叠与噪声混合音频

**—— 一份从问题发现到训练内化的完整实验报告**

---

## 摘要

大型音频-语言模型（Omni / Audio-LALM）在干净、单一说话人的语音上已表现优异，但在**多说话人重叠**、**环境噪声混合**等真实复杂声学场景下，"听清谁说了什么（who-said-what）"的能力显著下降。一个自然的增强思路是为模型提供结构化的上下文（context）。然而我们在实验初期发现一个关键陷阱：**当上下文中含有答案时，模型会直接"抄答案"而非"听音频"**，导致评测分数虚高（从 ~50% 飙升至 ~95%），这种提升经"静音探针"验证后被证明是泄漏而非真实理解。

为解决这一问题，本工作提出 **AGSC（Audio-Grounded Scaffold Context，音频接地的脚手架式上下文）**：一种"**只给线索、不给答案、零泄漏、可自动生成**"的上下文范式。其核心是只提供"谁在何时说话、背景多吵、哪里有语音、部分打乱的候选词"等导航性信息，并以"防抄契约 + 静音探针 + 泄漏门禁"三重机制保证非泄漏。我们进一步构建了**可泛化、可开源的合成流水线**（pyannote 说话人日志 + Mega-ASR 转写 + SepFormer 语音分离 + VAD/SNR 估计 + 门控），并设计了配套数据集与三种 Omni 模型（Qwen3-Omni-30B、MiniCPM-o-4.5-8B、Ming-flash-omni-2.0-104B-MoE）的 LoRA 微调方案。

在两类复杂场景、多个下游 benchmark 上的系统实验表明：（1）推理时注入 AGSC 在模型**真正失败的难样本**上带来一致正增益；（2）用 AGSC 数据做轻量微调后，模型对复杂音频的理解能力**实质跃升并内化**——重叠+噪声场景的 cpWER 从 26–71 降至 9–15，最难样本绝对改善 36–62 个百分点，且训练后"不给线索也变强"；（3）规律跨三模型一致，**越弱的模型收益越大**；（4）全程经静音探针与泄漏门禁验证为**非泄漏的真实增益**；（5）**实时性无实质代价**——AGSC 仅给 prompt 增加约 115–155 个文本 token，音频 token 主导 prefill，每次推理墙钟增量 ≤0.21 秒 / ≤12%（多数 ≤4%），且训练内化后可用 baseline 部署而零额外延迟。我们同时诚实记录了若干负结果（强模型在简单样本上线索成噪声、AST 声学场景标签不可靠、SAM Audio 不擅长同类人声分离等），并提供完整开源资料。

**关键词**：音频-语言模型；上下文增强；说话人重叠；噪声鲁棒；非泄漏评测；语音分离；LoRA 微调

## 1 引言与问题发现

### 1.1 背景与动机
Omni 模型能直接"听"音频并理解。但现实音频远不理想：会议里多人抢话、街头人声混在交通噪声里、远场带混响。在这些**复杂声学场景**下，模型常常分不清说话人、听错被噪声掩盖的词，"who-said-what"任务严重退化。

提升思路之一是给模型一份"参考资料"（context）：例如告诉它有几个人、各自说了什么。这能否真正帮助模型？这是本工作的出发点。

### 1.2 关键问题：分数虚高源于"抄答案"
我们最初按最朴素的方式实现（V1）：上下文中包含 XML 形式的完整标注（含每个说话人的转写文本）。结果惊人——三个模型加上下文后平均分从约 50% 升至约 95%（Qwen3 65.6→93.3、MiniCPM 53.4→96.8、Ming 40.1→91.1）。

但进一步分析揭示这是**假象**：
- **答案泄漏分析**：对 600 条子集逐任务统计"标准答案能否直接从上下文抄到"，转写/说话人归属等任务的可抄率高达 **~100%**。
- **污染测试（V2 Suite B）**：在上下文中注入**错误**标注，Qwen3/MiniCPM 在转写任务上"盲抄率（BCR）"高达 **94%–99.8%**——模型给错答案就跟着错，说明它在抄而非听。
- **静音探针（V3）**：把音频替换为**纯静音**，模型在含答案上下文下仍能答对（AC≈0 表示纯抄）。这是判定"抄 vs 懂"的决定性诊断。

**核心问题由此确立**：如何设计一种上下文，使得加上下文带来的提升**确凿地来自"听懂音频"，而非"抄答案/标签泄漏"**？这是后续全部工作的主线。

## 2 核心命题与方法学思想

> **核心命题**：为 Omni 模型提供高质量结构化上下文，能否真实提升其对复杂音频（多说话人重叠 / 环境噪声）的理解——且该提升来自"听懂音频"而非"抄上下文里的答案"？并做成可泛化、可开源的数据合成流水线。

围绕该命题，我们确立五条方法学原则：

1. **对症失败模式**：不给泛泛资料，而是针对模型在复杂音频上具体"栽跟头"的环节给线索（重叠 → 给说话人时间窗；噪声 → 给噪声档位/候选词）。
2. **零泄漏且可证**：每条线索过"泄漏门禁"（程序确保完整答案不出现在线索里）+ "静音探针"（去音频后若仍能答对则判为抄）。
3. **强前置决定线索质量**：线索由自动工具产出；**前置工具不够强 → 给错线索 → 误差累积反而伤害模型**。前置组件必须实测选优，而非看模型卡。
4. **难度切片看真相**：按信噪比（SNR）× 重叠程度 × 难易切片，重点观察模型**真正失败的难样本**——那里才是上下文应当起作用之处。
5. **从外部上下文到内在能力**：先在推理时证明线索有用，再用训练把它"训进"模型，并在**下游 benchmark** 上验证外部效度。

## 3 AGSC 上下文设计

### 3.1 设计原则：给线索不给答案 + 强制音频融合
AGSC 只提供**导航性线索**，绝不含完整答案：

- **重叠场景**：给每个说话人的**时间窗**（"目标说话人在 0.00–3.85 秒说话，性别=female"），不含其所说文字 → 帮模型在重叠中锁定并分离目标声源。
- **S1 单人+噪声**：给**噪声档位（吵/中/静，由 SNR 估计）+ 语音活动区（VAD）+ 部分打乱的候选词**（真词+干扰词，仅保留约 2/3 并打乱；数字等原子答案直接不给）。
- **S2 重叠+噪声（最难）**：先用语音分离器把两人拆开，各自 ASR 出草稿，再**只给每人"部分打乱的关键词"**（不给整句）→ 既提示"谁大概说了哪些词"，又无法照抄。

### 3.2 三重非泄漏保障
1. **防抄契约（Audio-First Contract）**：在提示中显式声明"线索由自动工具产生、未经验证、可能含错误与干扰项、不含完整答案、以音频为准、不要照抄"。
2. **泄漏门禁**：程序化校验——归一化后的完整答案不得作为线索内容的连续子串出现；候选词混入干扰、打乱顺序、仅保留部分。
3. **静音探针（Silent Solvability, SS）**：以静音替换音频，若模型在仅有线索时仍能答对，则判为泄漏；真实增益要求 SS 高（去音频后答不出）。

### 3.3 门控（Gating）—— 实证逼出的关键设计
我们发现：在**干净、无重叠、高 SNR**的样本上，强行注入线索反而引入噪声、**有害**。因此流水线自带门控：**仅当检出重叠 / 多说话人 / 低 SNR 时才注入线索**，否则退回纯音频。门控信号本身来自流水线（overlap 检出 / SNR 估计），可直接部署。实证表明门控能把整体增益从无脑全注入的 +1.2 提升到 +3.8（约 3 倍）。

## 4 数据合成流水线（可泛化 / 可开源）

`context_synth_pipeline.py` 是本工作的核心交付物，输入任意音频，自动产出零泄漏的 predicted-AGSC：

```
任意音频
 → [1] 说话人日志 (pyannote-3.1)        # 谁/何时说话
 → [2] 分段转写 (Mega-ASR)              # 每段大概说了啥（仅候选）
 → [3] 重叠检测 / VAD语音区 / SNR档位     # 噪声场景新增
 → [4] 重叠+噪声: SepFormer 分离两人 → 各自ASR → 部分打乱关键词
 → [5] 门控: 仅"重叠/低SNR"注入, 干净不注入
 → [6] 零泄漏 predicted-AGSC 导出 (+置信标注)
```

### 4.1 组件选型（全部实测，而非看模型卡）
- **说话人日志 = pyannote-3.1**：真实样本实测 DER 0.22 / 说话人数命中 75%，完胜 CAM++（DER 0.47 / 0%）。
- **ASR = Mega-ASR**：噪声样本 -5dB 实测 CER 0.05，抗噪 SOTA。
- **VAD = silero-vad**：稳定定位语音段。
- **SNR 估计**：基于 VAD 的能量比；点估误差偏大（MAE ~9.9 dB），但**粗档位命中率 79%**，仅用粗档。
- **声学场景标签 = AST(AudioSet)**：实测**场景精度仅 29%**（人声干扰 + 合成背景难），**按"宁缺毋滥"原则弃用**——这是一个被实证否决的组件，体现了"前置不可靠则有害"的原则。
- **语音分离 = SepFormer-whamr**（见 §10 的实证对比）。

### 4.2 Stage A 链路泛化实测
在 12 条真实样本上端到端跑通流水线并与 gold 对比：DER 0.15、说话人数命中 83%、分段 CER 0.20、置信标注分布 high 9 / medium 3 / low 0。难例（重叠极重 / 单人被误切两人）被置信规则正确降级为 medium，可用于下游过滤。

## 5 数据集构建

所有数据均**带真值、零泄漏**：

| 用途 | 数据 | 规模 | 真值 | 线索来源 |
|---|---|---|---|---|
| 重叠训练 | `stage_c_train_v2.jsonl` | 4728 | gold/双音频目标 | 门控 AGSC |
| 噪声训练 | `stage_n_train.jsonl` | 3500 | gold | 门控 AGSC（掺旧重叠防遗忘）|
| 合成重叠 | `agsc2.jsonl` / `agsc_tasks.jsonl` | 404 / 539 | gold | 时间窗/候选词 |
| S1 单人+噪声 | `speech_env`(eval 200/train 2400) | 4500源 | 转写+SNR+环境 | 噪声档位+候选词 |

**S2 的可控构造**：`noise_inject_s2.py` 将已有的 2 人重叠语音（SparseLibriMix2）与真实 WHAM 噪声按 {0,5,10} dB SNR 混合，**保留每个说话人的标准转写**，得到完全可控的"重叠+噪声"数据。带噪后 pyannote 仍能检出 2 人 + overlap（命中 81%，仅比干净 86% 略降），说明声源导航线索在噪声下仍可用。

## 6 任务与评测指标

- **转写类**：cpWER（2 说话人置换不变词错误率）、WER（目标说话人）、CER（中文字符级），越低越好。
- **分类类**：准确率（说话人计数、多说话人检测、谁说更久、性别异同、是否抢话、室内外等）。
- **判据**：`GCG = metric(NoCtx) − metric(+AGSC) > 0`（正=真实改善）；`SS = metric(+AGSC, 静音)` 高（非抄）；按 **SNR × 重叠 × 难易（按 baseline 三分位）** 切片。
- **下游 benchmark**（核实可用后纳入）：SparseLibriMix2、AMI TargetSpeaker-ASR、speech_env、SparseLibriMix2_noisy、NoisyLibriSpeech-MUSAN；外部效度补充 WildSpeech-Bench、MMAU、VoiceBench(噪声子集)、Dynamic-SUPERB Ph2。**诚实记录**：计划中的 "Audio MultiChallenge" 经核实在 HF 上无音频版本（仅 ScaleAI 纯文本），予以排除。

## 7 模型微调方法

采用 LoRA 轻量微调（仅注意力投影，约 0.05% 参数），三模型各自适配其异构接口：

- **Qwen3-Omni-30B**：对 thinker（真正的 LM）注入 LoRA；**DeepSpeed ZeRO-2 四卡数据并行**。标签仅监督 assistant(target)，prompt 与音频占位 mask 为 -100。
- **MiniCPM-o-8B**：复刻其 `model.chat` 的 teacher-forcing 输入构造，`forward(data, labels=)` 透传到底层 LLM 出 loss；单卡。
- **Ming-flash-104B-MoE（BailingMM2）**：复用其适配器 load（device_map 四卡 naive-MP），手动构造输入并直接 forward LLM；攻克 7 个自定义接口坑（模板只接受 HUMAN 轮、PEFT task_type、whisper 音频 dtype、cache bool、fused-CE 的 inplace_backward 等）。

训练目标统一：在门控 AGSC + gold 数据上学会"用线索在重叠/噪声中归属说话人并转写、且无线索时仍能转写"。

## 8 实验一：多说话人重叠场景（Stage A→B→C）

### 8.1 Stage B：下游验证（训练前，predicted-AGSC，零泄漏）

| 模型 | SparseLibriMix2 Δ全部 | 难1/3 Δ | AMI 真实会议 Δ全部 | AMI 难1/3 Δ |
|---|---|---|---|---|
| Qwen3 | +1.3 | +8.3 | +12.0 | +21.8 |
| MiniCPM | +15.8 | +36.1 | +6.0 | +2.9 |
| Ming | +0.8 | +12.8 | −1.7 | +3.8 |

**规律**：6 个高难（模型,任务）组合的**难1/3 增益全部为正**（+2.9~+36.1）；模型越弱、重叠 baseline 越差，可得增益越大。干净对照任务（MultiSpeakerDetection 98.8% 天花板、SpeakerCounting 工具≈模型同样无能）无增益——AGSC 只在"工具对、模型错"处有价值。**铁证**：AMI 上 baseline 把整段会议所有人都转写（过度转写），AGSC 给目标时间窗后模型聚焦目标（52 词→8 词），线索零转写，改善只能来自"听音频+用时间窗定位"。

### 8.2 Stage C：训练内化（held-out 同集前后对比，cpWER↓）

| 模型 | SparseLibriMix2 +AGSC 前→后 | AMI 会议 +AGSC 前→后 |
|---|---|---|
| Qwen3 | 25.3 → **9.0** | 69.8 → **53.3** |
| MiniCPM | 48.3 → **13.8** | （未训 AMI 双音频）|
| Ming | 67.4 → **7.0** | — |

训练后**不给线索（baseline）也大幅变好**（Qwen3 26.6→8.4、MiniCPM 64.1→14.9、Ming 68.3→7.9），能力被内化；越弱的模型降幅越大（Ming −60.4 > MiniCPM −34.5 > Qwen3 −16.3），三者收敛到 ~7–14。

## 9 实验二：环境噪声混合场景（Stage N-A→N-D）

### 9.1 Stage N-A：噪声组件实测取舍
新增并实测：VAD（采用）、SNR 档位（79% 命中，采用）、AST 场景标签（29%，**弃用**）、候选词偏置（零泄漏门禁，采用）。验证了"前置不可靠则有害"的原则。

### 9.2 Stage N-C：推理前下游验证（三种 S2 线索设计对比，Qwen3，cpWER↓）

| S2 线索设计 | 全部 Δ | 难1/3 Δ | 说明 |
|---|---|---|---|
| 弱前置：带噪 pyannote 光秃时间窗 | −1.6 | −2.0 | 噪声退化日志，窗不准→误导 |
| 强前置但坏设计：denoise+ASR 全草稿 | **−11.7** | −0.8 | 重叠转错的"流畅错草稿"强锚定→误差爆炸 |
| **强前置+好设计：SepFormer 分离+门控关键词** | −1.8 | **+1.3** | 避免灾难、难样本转正、零泄漏 |

![同场景三种 Context 设计的增益对比（Qwen3, S2）](results/fig_s2_design_comparison.png)

这组对比**实证了核心命题方法学的两点**：①前置工具强弱决定线索质量（错草稿导致 −11.7 灾难）；②上下文设计的精准性决定成败（同样强前置，"全草稿"vs"门控关键词"差异巨大）。

**三模型 N-C（最优 SepFormer 门控前置）**：

| 模型 | S1 全部Δ | S2 全部Δ | S2 低SNR Δ | S2 难1/3 Δ |
|---|---|---|---|---|
| Qwen3 | +0.2 | −1.8 | −1.5 | +1.3 |
| MiniCPM | +0.1 | **+3.2** | **+8.9** | **+15.7** |
| Ming | +0.0 | −1.0 | −1.5 | **+18.8** |

S2 难1/3 三模型全正（+1.3/+15.7/+18.8）= 线索在模型失败点有效；净增益正负取决于简单样本上不完美线索的噪声——这正是推理时增益不稳、需要训练内化的原因。

### 9.3 Stage N-D：训练内化（held-out 同集前后对比，cpWER↓）——决定性结果

| 模型 | baseline 前→后 | +AGSC 前→后 | 难1/3 前→后 |
|---|---|---|---|
| Qwen3 | 26.4 → **8.7**（−17.7）| 28.6 → **8.6**（−20.0）| 50.5 → **14.6**（−35.9）|
| MiniCPM | 65.8 → **14.0**（−51.8）| 62.6 → **22.5** | 70.6 → **18.1**（−52.4）|
| Ming | 71.1 → **15.0**（−56.1）| 72.1 → **14.7**（−57.4）| 77.6 → **15.4**（−62.2）|

![训练内化：三模型 S2 baseline cpWER 训练前后（左=全部，右=难1/3）](results/fig_training_internalization.png)

S1 单人+噪声：三模型本就近天花板（CER ~3-4%），训练后基本持平（±0.3）。

**解读**：训练把"在噪声+重叠中听清谁说啥"做实——S2 cpWER 从 26–71 全部降到 9–15，难样本降到 15–18；能力内化（训练后 MiniCPM/Ming 的 baseline 甚至 ≈ +AGSC，已不依赖线索）；越弱的模型收益越大。

## 10 前置分离器选型实证（SepFormer vs SAM Audio）

应用户建议，我们引入 Meta 的 **SAM Audio**（Segment Anything for Audio，按文本/视觉/时间提示分离任意声音）作为候选前置。在严苛工程条件下（服务器驱动 470，仅支持 CUDA ≤11.4），我们通过**纯环境侧手段让 SAM Audio 在老驱动上跑通**：torch 2.5.1+cu118（满足其 flex_attention/torch≥2.5 需求）、给其依赖的 torchcodec 做 soundfile 打桩（绕过对 CUDA-13 libnvrtc 的硬依赖）、关闭 cuDNN 与 torch.compile 走 eager（绕过 Triton 在老驱动上的无效 kernel），并 stub 掉 ImageBind/CLAP reranker，共约 30 余处依赖/兼容修复。

**实测对比**（同 12 个 S2 样本，分离→ASR 草稿 cpWER↓，越低=分离越准）：

| 前置 | cpWER | 结论 |
|---|---|---|
| **SepFormer-whamr** | **16.8%** | ✅ 采用 |
| SAM Audio base | 43.5% | ✗ |
| SAM Audio small-tv | 55.8% | ✗ |

**结论**：SAM Audio 是"按声音类型/描述分离任意声音"的通才（事件提取强），但**对两个同为人声的说话人分离不在行**（即便用时间窗 span 提示也分不干净）；who-said-what 任务上专为"2 人+WHAM 噪声"训练的 SepFormer 显著更优。这是一次"数据说话"的选型——我们确实下载、跑通、实测了 SAM Audio，最终基于证据选择 SepFormer。

## 11 结果综合与发现规律

贯穿重叠与噪声两阶段、三个模型，我们观察到四条稳定规律：

1. **增益集中在模型失败点**：所有高难（模型,任务）组合的难样本切片增益均为正。线索（时间窗/关键词）帮模型在重叠/噪声中分离与归属——这是其本来做不到、有线索+听音频后做到的真实多模态融合。
2. **训练内化**：训练后不给线索也大幅变好；弱模型训练后 baseline ≈ +AGSC，已不依赖推理时线索。上下文被训练成模型自身能力。
3. **越弱的模型收益越大**：弱模型（Ming/MiniCPM）训练后降幅最大，三者殊途同归收敛到接近水平。
4. **全程零泄漏**：线索不含答案，静音探针与泄漏门禁守住——增益是真实理解而非抄答案。

## 12 推理实时性与延迟开销

一个实用性问题：加入 AGSC 上下文会不会显著拖慢推理、影响实时性？我们对三个模型 × 两类场景做了实测（每组 n=20 样本，预热 2 次后计时，greedy 解码，max_new_tokens=200；端到端 e2e 含音频处理+prefill+decode，并用包裹 `generate` 捕获输入/输出 token 数）。

### 12.1 每次推理延迟：baseline vs +AGSC（端到端均值，秒）
| 模型 | 场景 | baseline | +AGSC | Δ 绝对 | Δ 相对 | 输入token base→agsc | 输出token base→agsc |
|---|---|---|---|---|---|---|---|
| Qwen3-Omni 30B | S1 单人+噪声 | 3.16 | 3.28 | +0.11 | **+3.6%** | 129→244 (+115) | 11.0→10.8 |
| Qwen3-Omni 30B | S2 重叠+噪声 | 6.54 | 6.50 | −0.04 | **−0.6%** | 192→347 (+155) | 33.2→32.8 |
| MiniCPM-o 8B | S1 单人+噪声 | 0.50 | 0.54 | +0.05 | **+9.1%** | 92→206 (+115) | — |
| MiniCPM-o 8B | S2 重叠+噪声 | 0.96 | 0.96 | −0.00 | **−0.5%** | 146→301 (+155) | — |
| Ming 104B-MoE | S1 单人+噪声 | 1.74 | 1.95 | +0.21 | **+11.9%** | 172→286 (+114) | 9.2→9.2 |
| Ming 104B-MoE | S2 重叠+噪声 | 2.50 | 4.23 | +1.73 | **+69%** | 270→427 (+157) | 14.1→26.7 |

![实时性：三模型×两场景端到端墙钟对比](results/fig_latency_overhead.png)

### 12.2 三条解读
1. **上下文本身只增加固定 ~115（S1）/ ~155（S2）个文本 token**。音频 token 主导 prefill，因此这点文本增量在总 prefill 中占比很小——当输出长度不变时（Qwen3 两场景、MiniCPM 两场景、Ming S1），每次推理墙钟变化落在 **±0.21 秒 / ≤12%** 区间，绝对增量都 ≤0.21 秒，S2 上甚至因测量噪声呈微负。结论：**AGSC 的 prefill 开销可忽略，不影响实时性**。
2. **唯一显著的 Ming S2（+69%）不是上下文开销，而是"模型把活干完了"**：其输出 token 从 14.1 翻倍到 26.7——在线索帮助下 Ming 真正把两个重叠说话人**都**转写出来（baseline 会塌缩成一路、欠转写）。多花的时间来自 decode 出更多正确内容（对应该格 cpWER 的大幅改善），是"做对事"的代价，而非"上下文税"。延迟随输出长度自回归增长，与输入是否带线索无关。
3. **相对涨幅在小模型上更醒目只因基数小**：MiniCPM/Ming 单次本就 0.5–2 秒，固定 prefill 增量占比自然更大，但绝对值仍是几十~两百毫秒量级。

### 12.3 离线线索生成成本（与在线推理解耦）
线索由**离线流水线一次性预生成**，不进入在线推理路径。实测 S2 线索生成（SepFormer 分离 + Mega-ASR 双路转写 + 门控关键词）：模型加载一次性 19.4 秒；**每样本均值 1.56 秒**（分离 0.21s + 双路 ASR 1.33s，中位 1.23s）。S1 线索更轻（VAD+SNR 档位+候选词，无分离）。该成本可批量化、可缓存，属数据准备阶段，与在线服务延迟无关。

### 12.4 部署建议（实时性最优解）
**训练内化后 baseline ≈ +AGSC**（弱模型尤其明显，见 §9.3），意味着线上**可直接用 baseline prompt 部署——零额外推理延迟，却保留训练带来的精度增益**。推理时线索只需在门控判定的难场景（重叠/低 SNR）按需注入；即便注入，prefill 增量也可忽略。因此 AGSC 在实时性上**无实质代价**：要么训练内化后零开销，要么推理时注入也仅毫秒级 prefill，唯一较大的耗时（Ming S2）本质是模型产出更完整正确结果所需。

## 13 流式场景扩展：增量 Context 与实时门控

将音频改为流式输入（t 时刻仅能听到并处理前 t 秒），在 S2（重叠+噪声）上系统考察两个问题：①Context 在短前缀音频时是否已有增益；②实时门控注入是否能"干净段不打扰、复杂段上脚手架"。（详见《流式场景_完整报告.md》；55 条≥8s 样本×7 前缀×3 条件×3 模型 + 30 条合成流×5 策略 + 360 个一 token 探针窗。）

### 13.1 增量增益曲线（前缀线索仅由 audio[0:t] 算出）
线索信息量随音频积累：1s=0 关键词 → 3s≈5 → 8s≈15 → 全量≈17。两条曲线分离混杂因素：曲线A=截断音频+前缀线索（真流式）；曲线B=完整音频+前缀线索（线索成熟度）。

![难1/3 切片曲线（曲线A）：绿色阴影=流式线索在失败点的增益；全样本均值因简单样本稀释而显平，明细见 stream_curve_table.md](results/stream_hard_curve.png)

![曲线B 线索成熟度（难1/3）：三模型增益线全程为正、随线索信息量（灰柱，右轴）同步增长](results/stream_hard_gain_info.png)

核心结果（GCG，pp，正=有益）：**1–2s 无增益甚至负增益**（线索物理上无信息，Ming 1s 处 −11.2）；**3–4s 增益窗口打开**（曲线A：Qwen3 全部 +2.9、难1/3 +8.2@3s）；**弱模型难样本增益随线索成熟稳定增长**（曲线B 难1/3：Ming 2s +15.1 → 全量 +21.2，6s 已达全量约 79%；MiniCPM 最高 +15.7@8s）。结论：**Context 增益不必等 offline 全量音频，但需 3–6s 积累让线索"成熟"**；成熟后流式与 offline 同规律（弱模型+难样本受益最大）。

### 13.2 实时门控注入（30 条 [干净4s|复杂4s|干净3s] 合成流）

![门控五策略：上排 cpWER（黑框=最优），下排干净段召回（虚线=基线）——注意力虹吸与 gated_v2 修复](results/fig_gate_strategies.png)

- **信号检测器**（逐 1s 窗 pyannote 重叠+滞回平滑）：逐秒 F1 0.53，触发延迟 1.72s，干净段豁免率 75%，注入量省 65%。
- **端到端五策略**发现并修复**注意力虹吸**：带时间窗的门控线索使模型聚焦复杂段、干净段召回暴跌（Qwen3 97.6→73.5）；线索中加一句"仍需完整转写全程"（gated_v2）后 Qwen3 干净段召回恢复 97.6 且整体 cpWER 全场最优（36.2 vs 基线 38.1）。门控收益模型相关：MiniCPM 上 oracle 门控整体 −7.6；Ming 对混拼流注入敏感、全策略略负（与其 offline 规律一致）。检测器质量是门控上限（gated 与 oracle 窗差 5.8–6.5pp）。
- **一 token 探针**（模型自身当门控开关，零样本）：CLEAN 识别 90–100%，COMPLEX 仅 0–57%（边界保守，Ming 几乎全报 CLEAN）；2s 窗显著优于 1s 窗；MiniCPM 单窗延迟 0.30s 证明"小模型哨兵+大模型干活"实时可行。该结果给出 GRPO 内化门控的冷启动下界：方案为 `<ctx_on>/<ctx_off>` 策略 token + 组内相对优势奖励（r=下游 cpWER 改善 + α·门控 F1 − λ·注入成本），合成流真值免标注，留作下一阶段。

### 13.3 与 offline 四规律的对照
增益集中失败点 / 前置+设计决定成败 / 弱模型收益大 / 干净场景需门控——四条在流式设定下**全部复现**；新增第五条：**线索措辞的一句话（"仍需完整转写"）可改变门控成败**，Context 工程精准度的又一实证。

### 13.4 GDPO 强化学习内化门控（三模型完整验证）

用 GDPO（arXiv:2601.05242，多奖励组内**解耦归一化**）把门控决策训成模型自身能力（详见《GDPO全阶段强化学习_论文级报告.md》）。方法：二元动作 {COMPLEX→注入, CLEAN→不注入} 上的解析策略 + ε-探索行为策略 + PPO clip；双奖励（门控正确性 + 实测下游收益查表，干净段误注入 −0.5）；LoRA，120 训练流/240 窗，评测全 held-out。

![GDPO 三模型结果：探针 COMPLEX 召回 / 端到端 F1 / 推理迁移](results/fig_gdpo_results.png)

| 模型 | 探针 COMPLEX 识别 | 端到端时间线 F1 | 触发延迟 | 听觉计数推理 acc |
|---|---|---|---|---|
| Qwen3 | 0.567 → **1.00** | 0.727 → **0.803** | 0.73→0.20s | 16.3% → **26.3%** |
| MiniCPM | 0.567 → 0.70 | 0.720 → **0.769** | 0.53→0.17s | 35.0% → 35.0% |
| Ming | 0.20 → **1.00** | 0.522 → **0.753** | 1.19→0.23s | 15.0% → **21.2%** |

四个发现：①三模型门控 F1 全部超过外置信号检测器（0.75–0.80 vs 0.53），**"模型自发 `<ctx_on>`"路线被证实**；②受控对照下 **GDPO ≻ GRPO**（训练达 ε-探索上限 0.875 vs 平台 0.75–0.80、GRPO 首轮出现一边倒塌缩）——论文"解耦归一化保留多奖励信号"在真实多模态任务复现；③**推理智商迁移**：只训二元门控，独立的听觉计数推理基准 Qwen3 +10pp / Ming +6.2pp——方法链提升的不只是 WER，还有声学场景元认知；④越弱收益越大再现（Ming 探针 +37pp）。诚实记录：Ming 召回↑伴随精确率/干净段豁免率下降（cpWER 43.5→48.8），可由 GDPO 解耦权重 λ 调平衡；MiniCPM 推理基准持平（无灾难遗忘）。工程要点：RL 必须与部署同模板（SFT 扛得住模板错位、RL 扛不住）；chat 采样接口失效时二元动作可用解析策略+ε-探索替代 rollout 采样。

### 13.5 GDPO 全链路内化（Gate-then-Transcribe，三模型完整 before/after）

把重叠+噪声转写、门控决策、流式增量线索利用**统一进一次 GDPO 训练**：每回合=流音频（70% 复合/30% 纯干净）+ 随机成熟度线索（none/2s/4s/full），模型单次生成"GATE 行+转写"；三奖励（格式 / 门控 / **1−cpWER 真奖励**）解耦归一化；温度采样 G=4（修复 MiniCPM `do_sample=` 参数名后三模型采样全通）。160 条训练流、LoRA、1–2 epochs。（详见《GDPO全阶段强化学习_论文级报告.md》。）

![全链 GDPO：转写内化——不给线索也大幅变好](results/fig_chain_results.png)

核心结果（E2 S2 heldout 55 条，cpWER 全部/难1/3）：
| 模型 | 无线索 base→chain | 全量线索 base→chain | 推理基准 Δ |
|---|---|---|---|
| Qwen3 | 33.0→**30.2** / 67.6→**61.5** | 31.6→29.9 | SC +1.2pp, MSD +1.2pp |
| MiniCPM | 58.5→**44.4** / 86.1→**71.6** | 57.0→**42.9** | 持平（零遗忘） |
| Ming | 100→**35.8** / 100→**67.8** | 89.9→**38.4** | MSD **+5pp** |

结论：①**"不给线索也大幅变好"再次成为内化铁证**（仅 160 流×1-2 epoch 的 RL 数据量）；②Ming 零样本连输出格式都不会（遵循率 3.3%）——全链 RL 同时教会"怎么说"与"听什么"（E1 cpWER 97.5→35.4、干净段召回 3.3→99.4），**越弱收益越大第四次复现**；③各成熟度线索利用全面更娴熟（Qwen3 t4 难1/3 63.2→53.5）；④诚实记录与修复：MiniCPM 的 E1 门控在联合训练中被 asr 主导目标挤压（0.60→0.367，Qwen3/Ming 无此现象）；用 GDPO 自带机制修复（Eq.7 权重 1,2,2 + Eq.8 条件奖励 r_asr×𝟙[gate对]）后**门控 0.367→0.90（反超 base），t4 难样本门控+转写双优（64.3→54.1）**，代价是无线索纯转写内化部分回吐（−14.1→−3.0）——多目标张力被转化为解耦权重下的可控帕累托选择，实测验证了 GDPO 两个调节机制的有效性。

## 14 讨论

### 14.1 为什么有效
复杂音频的核心失败模式是"分不清声源 + 听不清词"。AGSC 的时间窗线索为模型提供"该听谁"的注意力支架，噪声档位/候选词提供"哪里有语音、可能涉及哪些词"的纠错先验。模型据此把多模态注意力对齐到目标，从而在重叠/噪声中完成本来做不到的分离与转写。训练则把这种"对齐与利用线索"的策略固化为内在能力。

### 14.2 边界与诚实记录的负结果
- **强模型在简单样本上线索成噪声**：Qwen3 在 S2 简单样本上本就会做，不完美线索反而轻微拖累，导致推理时净增益不稳。
- **干净 / 近天花板任务无空间**：S1 单人噪声转写、干净多说话人检测，模型本就强，AGSC 无增益。
- **前置不可靠则有害**：AST 场景标签 29% → 弃用；带噪 pyannote 光秃窗 → −1.6；错草稿 → −11.7。
- **SAM Audio 不适合同类人声分离**：通才模型在专门任务上不及专才。
这些负结果与方法学原则自洽，并非方法失效，而是界定了 AGSC 的适用边界（难重叠 / 低 SNR / 弱模型 / 训练内化）。

### 14.3 与"抄答案"的闭环
V1 含答案 → 抄答案虚高（已证）；V2/V3 用污染测试与静音探针证明"抄 vs 懂"；AGSC 给对症线索 → 在模型真正的失败点带来真实、非泄漏提升；Stage C/N-D 训练把能力内化。从"抄答案"到"真理解"的闭环完整。

## 15 工程实现与避坑（开源价值）

- **老驱动（470, CUDA 11.4）贯穿全程的墙**：所有 torch 必须 cu118；DeepSpeed 需在环境内装 CUDA 11.8 nvcc 并设 CUDA_HOME（否则其 FusedAdam 按系统 CUDA12.8 编译，生成 CUDA12 kernel 在 470 上报 "driver insufficient"）；SAM Audio 需 torch2.5+cu118 + torchcodec 打桩 + eager。
- **国内网络镜像**：conda=USTC、pip=Aliyun、github=ghfast.top、HuggingFace=hf-mirror.com。
- **依赖安全**：一律 `--no-deps`，防止某依赖把 torch 偷偷升级到 cu13/cu130 而破坏整个环境。
- **多环境隔离**：omni-context（Qwen3+DeepSpeed）/ omni-context-mcpm（MiniCPM）/ ming / omni-pipeline（流水线）/ samaudio-venv（SAM Audio）。

## 16 局限与未来工作

1. **S1 受天花板限制**：需更难的噪声集（更低 SNR、远场、非中文）才能显现增益。
2. **更难场景未覆盖**：3 人以上重叠 + 噪声、强混响、口音变化。
3. **外部效度可扩**：WildSpeech-Bench/MMAU/VoiceBench 的 judge 评分尚未全量接入。
4. **训练规模**：当前 1 epoch LoRA；更长训练、更大 rank 值得探索。
5. **GRPO 内化门控**：流式实验已给出完整设计（§13.2，`<ctx_on>/<ctx_off>` 策略 token + 程序化奖励 + 一 token 探针冷启动证据），训练为下一阶段最高优先级。
6. **流式前置低延迟化**：1–2s 超短前缀无增益的瓶颈在前置组件（SepFormer/ASR 需≥3s 积累），流式分离与流式 ASR 可前移增益窗口。
7. **置信感知线索**：根据前置组件置信度自适应调节线索强度。

## 17 结论与贡献

本工作系统验证了一个命题：**用"只给线索不给答案、零泄漏、可自动生成"的 AGSC 上下文，配合轻量训练内化，能让多个不同架构/规模的 Omni 模型在多说话人重叠与环境噪声混合的复杂场景下，对 who-said-what 的理解获得真实、可复现、抄不来的提升。**

主要贡献：
1. **AGSC 零泄漏线索范式 + 静音探针验证**：为"上下文增强"提供了判别"真懂 vs 抄答案"的方法学。
2. **可泛化/可开源合成流水线**：任意音频自动产出门控、零泄漏线索（pyannote+Mega-ASR+SepFormer+VAD/SNR）。
3. **强前置选型方法论**：以分离草稿 cpWER 实证对比（SepFormer vs SAM Audio），按数据决策。
4. **三种异构 Omni 模型的 LoRA SFT 训练器**（DeepSpeed/单卡/naive-MP，含多音频接口）。
5. **决定性实验结果**：重叠+噪声场景 cpWER 26–71 → 9–15，难样本绝对改善 36–62 个百分点，跨三模型一致、弱模型最大、全程非泄漏。
6. **实时性实测**：三模型×两场景的延迟对比表明 AGSC 在线推理开销可忽略（prefill 文本增量 ≤155 token、墙钟 ≤0.21s/≤12%），并给出"训练内化后零延迟部署"的工程结论。
7. **流式场景系统验证**：增量增益曲线（线索需 3–6s 音频积累成熟，6s 达全量增益约 8 成，弱模型难样本最高 +21.2pp）+ 实时门控端到端（发现并修复"注意力虹吸"，门控省 65% 注入量且 Qwen3 整体最优）+ 一 token 探针与 GRPO 内化门控设计。
8. **完整开源资料 + 老驱动避坑全集**，便于社区复现与合成更大规模的带有效标注的多模态音频数据集。
