# Omni-Context 总报告：用"只给线索不给答案"的上下文，真实提升 Omni 模型对复杂音频的理解

> 一句话：我们让"能听音频的大模型（Omni 模型）"在**多人抢话、嘈杂噪声、流式输入**这些真实复杂场景下，准确听清"**谁说了什么**"——做法是给模型一种**只给线索、不给答案、可自动生成、零泄漏**的上下文（**AGSC**），并通过**轻量训练把这种能力变成模型自己的本事**。在 **3 个不同架构/规模的 Omni 模型**（Qwen3-Omni 30B、MiniCPM-o 8B、Ming-flash 104B-MoE）× **3 类场景**（重叠、噪声+重叠、流式）上，拿到**真实、可复现、抄不来**的提升，并实测了**实时性代价（可忽略）**。

---

## 一、发现了什么问题

### 1.1 出发点
让大模型听复杂录音（多人同时说话、背景很吵）听清"谁说了什么"很难。直觉做法：给模型一份"参考资料"（context）帮它理解。

### 1.2 致命陷阱：加了 context 分数飙高，但是**抄来的**
按最朴素的方式做（参考资料里直接写答案），分数从 50% 飙到 95%，看着惊艳。一查是**作弊**：
- **答案泄漏**：参考资料里有答案，模型直接抄，根本没听音频。
- **测谎实验（静音探针）**：把音频换成**纯静音**，模型居然还答对——铁证它在抄，不在听。

**结论：分数高 ≠ 真听懂。整个项目要破的核心问题——让提升来自"听懂音频"，而不是"抄答案"。**

---

## 二、解决方案（核心思想）

### 2.1 AGSC：只给线索，不给答案
打个比方——帮朋友在吵闹派对里听清某个人说话：
- ❌ 错（抄答案）：直接告诉他"那人说的是『明天三点开会』"。他不用听了。
- ✅ 我们（给线索）：只告诉他"**你要听的人在第 3–6 秒说话**"或"**背景很吵，可能涉及这些词**"。**具体内容他还得自己竖起耳朵听。**

线索只含"谁在何时说话 / 噪声多大 / 哪里有语音 / 部分打乱的候选词"，**绝不含完整答案**——模型必须真听音频才能答对，想抄也抄不到。

### 2.2 三道保险（保证"真懂、非抄"）
1. **防抄契约**：明确告诉模型"线索可能错、不含答案、以音频为准"。
2. **静音探针**：去掉音频还答对 = 抄 → 用来验证非泄漏。
3. **泄漏门禁**：程序检查完整答案绝不出现在线索里（候选词只给一半且打乱、数字类答案不给）。

### 2.3 可开源的自动合成管线
```
任意音频 → 说话人日志(pyannote, DER 0.22) → 分段转写(Mega-ASR, -5dB CER 0.05)
        → 重叠检测/VAD语音区/SNR档位(79%准) → 重叠+噪声: SepFormer分离→各自ASR→部分打乱关键词
        → 门控: 仅"重叠/低SNR"时注入(干净时注入反伤) → 零泄漏 predicted-AGSC
```
前置选型全部实测说话：分离器 **SepFormer 16.8% ≫ SAM Audio 43–56%**（draft-cpWER）；AST 场景标签 29% 准确率→弃用。

## 三、结果一：训练内化——本项目最重要的图

![训练内化：三模型 S2 重叠+噪声 cpWER 训练前后对比（左=全部样本，右=最难1/3）](results/fig_training_internalization.png)

**怎么读**：灰柱=训练前不给线索的基线，绿柱=用 AGSC 数据做 LoRA 微调后**同样不给线索**的成绩（cpWER 越低越好）。
- **三模型从 26–71 全部压到 9–15**：Qwen3 26.4→8.7（−18）、MiniCPM 65.8→14.0（−52）、Ming 71.1→15.0（−56）。
- **最难 1/3 样本改善 36–62 个百分点**（右图）：原来基本听不清的录音（cpWER 50–78），训练后降到 15–18。
- **关键**：训练后**不给线索也大幅变好**——"会用线索听清"被训成了模型自身能力，这是抄不来的实质跃升（静音探针全程验证非泄漏）。
- **越弱的模型收益越大**：Ming（−56）> MiniCPM（−52）> Qwen3（−18），三者殊途同归收敛到 9–15。

重叠场景（无噪声）同规律：SparseLibriMix2 上 Qwen3 25.3→9.0、MiniCPM 48.3→13.8、Ming 67.4→7.0；真实会议 AMI 上 Qwen3 69.8→53.3。

## 四、结果二：前置与设计决定成败——同一场景三种做法天壤之别

![S2 三种 Context 设计对比（Qwen3）：弱前置误导、强前置坏设计灾难、强前置好设计难样本转正](results/fig_s2_design_comparison.png)

**怎么读**：纵轴是增益（正=有益）。同一个 S2 场景、同一个模型，只换线索的"产法和写法"：
1. **弱前置**（带噪 pyannote 的光秃时间窗）：窗不准→误导，全部 −1.6；
2. **强前置但坏设计**（降噪+ASR 整句草稿）：重叠转错的"流畅错草稿"强锚定模型→**灾难 −11.7**；
3. **强前置+好设计**（SepFormer 分离→门控打乱关键词）：避免灾难、**难样本转正 +1.3**、零泄漏。

**这就是"前置组件能力 + Context 设计精准度是决定性的"的直接证据**——线索质量不够时不是没用，是反伤。

推理时注入（不训练）的全景规律：增益集中在**模型真正失败的难样本**（所有高难组合的难1/3 增益均为正，最高 +36.1），强模型在简单样本上线索是噪声→所以需要门控。

## 五、结果三：实时性——加 Context 几乎不影响速度

![实时性：三模型×两场景端到端推理墙钟，baseline vs +AGSC](results/fig_latency_overhead.png)

**怎么读**：AGSC 只给 prompt 增加约 **115（S1）/155（S2）个文本 token**，而 prefill 的大头是音频 token，所以左右两图中灰绿柱几乎等高——**墙钟差 ≤0.21 秒（多数 ≤4%）**，部分场景因测量噪声还略快。
- 唯一明显的 **Ming-S2（2.50→4.23s，+69%）不是"上下文税"**：它的输出 token 从 14 翻倍到 27——在线索帮助下终于把两个重叠说话人**都**转写出来（基线会塌缩成一路）。多花的时间是"把活干完"的代价。
- 线索由**离线管线预生成**（S2 实测 1.56s/样本：分离 0.21+双路 ASR 1.33），不进在线推理路径。
- **部署最优解**：训练内化后 baseline≈+AGSC → 线上直接用 baseline prompt，**零额外延迟保留全部精度增益**。

## 六、结果四：流式场景——线索要"长"几秒？门控注入靠谱吗？

### 6.1 增量增益曲线：线索需要 3–6 秒"生长期"

![难1/3 切片：绿色阴影=流式线索在模型失败点的增益区（Ming 从 3s 起 −3.4→−12.3）](results/stream_hard_curve.png)

**怎么读**：模拟流式输入——t 时刻模型只能听到前 t 秒、线索也只能由前 t 秒算出；图中只看每个模型最难的 1/3 样本（模型真正失败的地方，这是本方法的目标场景）。**绿色阴影=线索带来的增益区**：
- **1–2s 无增益**：1s 音频算不出关键词（信息量=0）、2s 仅约 2 个——线索物理上还没"长出来"，不是方法失效，是信息边界。
- **3s 起增益打开**：Qwen3 在 3s 处 −8.2（它只听到 3 秒、自身 cpWER 88 时，5 个关键词拉低 8 个点）；**Ming 从 3s 起整片绿色增益区**，−3.4 → −8.3 → 全量 −12.3。
- 全样本均值上增益会被"简单样本上线索是噪声"平均掉（这正是需要门控的原因），故均值曲线不入正文，明细见 `results/stream_curve_table.md`。

![难样本增益随线索成熟而增长：三条增益线全程为正，与线索信息量柱（右轴）同步上行](results/stream_hard_gain_info.png)

**怎么读**：把"音频没听全"和"线索没算全"两个因素拆开——模型听完整音频，只让线索"分批到货"；纵轴是难1/3 上的增益，灰色柱是线索关键词数（右轴）。**三条增益线全程在零线上方**，且 Ming > MiniCPM > Qwen3（越弱收益越大）。关键读数（图中箭头标注）：**Ming 用 2 秒音频算出的线索已 +15.1，6 秒拿到全量增益的约 8 成**（全量 +21.2）。**结论：流式不必等 offline 全量音频，给线索 3–6 秒生长期即可。**

### 6.2 实时门控：可行、省 65% 注入量，但"一句话措辞"决定成败

![门控五策略对比：上排 cpWER（黑框=最优），下排干净段召回（虚线=基线）](results/fig_gate_strategies.png)

**怎么读**：30 条 [干净4s|复杂4s|干净3s] 合成流上比五种策略。三个故事：
1. **注意力虹吸（重要新发现）**：下排 gated/gated_gt 的矮柱——朴素的"4–8s 检测到复杂段"时间窗线索把模型注意力吸进窗口，**干净段被静默漏转**（Qwen3 召回 97.6→73.5，MiniCPM 最惨 →57.3）。
2. **一句话修复**：线索里加"仍需完整转写全程"（gated_v2，绿柱）→ Qwen3 干净段召回弹回 97.6、整体 cpWER 36.2 **全场最优**（基线 38.1）。Context 措辞的一句话=几个点的 cpWER。
3. **门控收益模型相关**：MiniCPM 上 oracle 门控整体最优（−7.6）但伤干净段；Ming 对混拼流任何注入都略负（同 offline 规律——它的收益在难样本和训练内化）。
- 信号检测器（逐秒 pyannote 重叠+滞回）：F1 0.53、触发延迟 1.72s、**干净段豁免率 75%、注入量省 65%**；检测器质量是门控上限（与真值窗差 ~6pp）。
- **模型自己当开关（一 token 输出 CLEAN/COMPLEX）**：零样本下 CLEAN 近满分、COMPLEX 仅 0–57%（不敢报警）——表征里有信号、边界保守；MiniCPM 单窗 0.3s，实时性达标。
- **强化学习已补上这块短板（GDPO，三模型完整验证）**：用"门控正确+实测下游收益"双奖励的 GDPO（解耦归一化，arXiv:2601.05242）训练后，**三模型 COMPLEX 识别 56.7/56.7/20 → 100/70/100**，模型当流式检测器的 F1 全部反超外置信号检测器（0.75–0.80 vs 0.53）、触发延迟降到 ~0.2 秒；同条件对照证明 GDPO 优于 GRPO（多奖励信号不塌缩）；**还白赚了推理智商**：只训门控，听觉计数推理基准 Qwen3 +10pp、Ming +6.2pp——方法提升的不只是"听写"，还有"听懂场景"（详见《GDPO全阶段强化学习_论文级报告.md》）。

![GDPO 三模型结果：探针识别 / 检测器F1（红虚线=信号检测器）/ 推理迁移](results/fig_gdpo_results.png)

- **终极一步：全链路一锅训（GDPO 全链内化）**：把"听重叠噪声、判断何时要线索、用好不同成熟度的流式线索"三件事放进**一次强化学习**（模型单次生成"GATE 判断行+转写"，奖励=格式+门控+真实 cpWER）。结果：**不给线索推理时 cpWER 也大降**——Qwen3 33→30、MiniCPM 58.5→44.4、**Ming 100→35.8**（它本来连输出格式都不会，RL 一并教会）；各档线索利用全面更好；Ming 场景判断推理还 +5pp。诚实记录：MiniCPM 在联合目标下门控小幅退步（多目标挤压，可调权重修复）。**整条链路的能力，确实能内化进模型内部。**

![全链 GDPO 内化：不给线索也大幅变好（右图=最难1/3）](results/fig_chain_results.png)

## 七、四条铁律（贯穿重叠、噪声、流式三大场景）
1. **增益集中在模型失败点**：所有高难组合的难样本切片增益全为正；简单样本上线索是噪声→需门控。
2. **训练内化是主增益**：训练后不给线索也大幅变好（cpWER 26–71→9–15），Context 变成模型自身能力。
3. **越弱的模型收益越大**：推理时和训练后均成立（Ming/MiniCPM 降幅最大）。
4. **前置能力+Context 设计精准度决定成败**：弱前置/坏设计/坏措辞会反伤（−11.7、注意力虹吸），强前置+好设计+好措辞翻盘（+1.3、36.2 全场最优）。
全程零泄漏：静音探针 + 泄漏门禁守住——增益是真实理解而非抄答案。

## 八、诚实记录的负结果（界定适用边界）
- 干净/近天花板任务（S1 单人噪声 CER~3%、干净多说话人检测）无增益空间；
- 强模型简单样本上线索成噪声（Qwen3 S2 推理时整体 −1.8）；
- AST 场景标签 29% 不可靠→弃用；SAM Audio 不擅长同类人声分离（43–56% vs SepFormer 16.8%）；

- gated_v2 措辞对 MiniCPM 有副作用（话多→插入错误）——没有万能 prompt；
- Ming 在干净/复杂混拼流上任何注入都略负——其价值在难样本与训练内化。

## 十、门控塌缩攻坚：当"开关"卡死后怎么救回来

### 10.1 问题：训练后模型变成"线索话痨"
三个模型大训练后都犯了同一个毛病：我们设计的"门控开关"（让模型先判断"这段音频复杂吗"，复杂才注入线索）**全部卡在常开**——连干净的单人录音也判成"复杂"。干净流门控准确率(gate_acc)：Qwen3、MiniCPM 掉到 0，Ming 只剩 0.11。根因是训练集里干净样本占比太低，模型学懒了：反正全判"复杂"也不吃亏。

### 10.2 第一条路（B 软门控）：失败，但失败得有价值
对标 IRAF 的"连续概率门控"，我们猜：模型输出 COMPLEX/CLEAN 前其实有个连续置信概率 π，也许 argmax 卡死了但概率还有区分度，按概率分三档注入就能救？

做了 π 探针（forced-choice，直接读模型对 COMPLEX/CLEAN 两个词的概率）。一盆冷水：训练后的模型，**干净流和复杂流的 π 都饱和到 1.0**（完全分不开）。而没训练的 base 模型，干净流 π 只有 0.012、复杂流 0.915（区分得清清楚楚）——证明探针没问题，是训练把概率"压死"了。结论：连续概率也塌缩了，软门控三档阈值退化成一档，**救不了**。这是个有价值的负结论：连续门控要成立，前提是概率有"梯度可达性"。

### 10.3 第二条路（A-RL 续训）：也失败
那就加大门控奖励、补上干净样本继续强化学习续训？跑了 76 步，门控奖励卡在 0.5（瞎猜水平）上不去，其中 18 步梯度直接是 0。原因很深刻：概率已经饱和（两个词 logit 差了 20），softmax 梯度趋近 0，强化学习的信号传不回去；加上 GDPO 组内归一化，几个采样常常同向、优势互相抵消。**强化学习救不动已经饱和的门控。**

### 10.4 第三条路（A-SFT 监督微调）：破饱和，成功
转念一想：RL 在饱和点没梯度，那监督学习正相反——交叉熵 −log p(CLEAN) 在 p 越接近 0（越饱和）时 loss 越大、梯度越猛，正好能"撬开"饱和。做法：拿干净样本的金标 "GATE: CLEAN" 做监督微调，干净样本上采样到一半。
- **第一版踩坑**：只教"GATE: CLEAN"一行，门控治好了（gate_acc 0→1.0），但模型学会"说完开关就闭嘴"，转写整个没了（cpWER 飙到 100）！
- **第二版修复**：金标改成"GATE 行 + 完整转写"，让模型学会"判完开关继续转写"。两全其美。

### 10.5 结果：三模型门控全部治住，转写还更好
| 模型 | 干净流门控 gate_acc | mix 无线索 cpWER↓ | 复杂段召回↑ |
|---|---|---|---|
| Qwen3 | 0 → **1.0** | 15.8 → 14.6 | 80.3 → 81.5 |
| MiniCPM | 0 → **1.0** | 20.3 → 21.6 | 71.3 → 77.7 |
| Ming | 0.11 → **1.0** | 19.0 → 17.5 | 74.6 → 80.9 |

门控全部从塌缩恢复到满分，转写不仅没退化、大多还改善了（监督用的是金标转写，顺带强化了 ASR）。详见 `results/fig_gatefix.png`。
