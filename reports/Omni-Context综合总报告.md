# 摘要

让「能听音频的大模型」（Omni 模型）在多人抢话、嘈杂噪声、流式输入这些真实复杂场景下准确听清「谁说了什么」（who-said-what），是当前音频大模型的一大短板：它们在干净单说话人语音上表现优异，一旦遇到多说话人重叠、环境噪声混合、远场混响，转写质量与说话人归属能力会显著退化。

本报告汇总一条完整的研究链路，回答一个核心命题并把它做实：**能否为 Omni 模型提供一种「只给线索、不给答案、零泄漏、可自动生成」的上下文（AGSC），真实提升它对复杂音频的理解——且提升确凿来自「听懂音频」而非「抄上下文里的答案」，并最终把这种能力内化成模型自身的本事。**

研究分五个递进的阶段展开：

1. **重叠场景**——确立 AGSC 范式，证明在多说话人重叠下线索能在模型失败点带来真实增益，并通过训练把增益内化；
2. **噪声场景**——在重叠之上叠加环境噪声（最难场景），证明「前置组件能力 + 上下文设计精准度」是决定成败的关键变量；
3. **流式场景**——把音频改成流式输入，回答「线索要积累几秒才有用」「能否实时门控」两个部署关键问题；
4. **强化学习全链路内化**——用一次 GDPO 强化学习把「门控决策 + 重叠噪声转写 + 流式线索利用」整条链路内化进模型；
5. **门控塌缩攻坚**——诊断并矫正强化学习训练诱导的一个系统性副作用，最终让三个模型的门控能力统一恢复。

**核心成果一览**：

- **诊断并破解「抄答案」陷阱**：含答案的上下文使分数从约 50% 虚高到约 95%，三项诊断（泄漏分析、污染测试、静音探针）证明这是泄漏而非理解；改用零泄漏 AGSC 线索后，提升确凿来自「听懂音频」。
- **重叠 / 噪声场景训练内化**：重叠+噪声的 cpWER 从 **26–71 全部压到 9–15**，最难 1/3 样本绝对改善 **36–62 个百分点**，训练后「不给线索也大幅变好」，跨三模型一致、越弱收益越大、全程经静音探针验证非泄漏。
- **流式场景系统验证**：线索需 **3–6 秒成熟**（6 秒音频算出的线索已拿到全量约 8 成增益），实时门控省 **65% 注入量**，并发现与修复了「注意力虹吸」现象。
- **强化学习全链路内化**：仅用监督微调五分之一的数据量（160 条流）把整条链路内化，无线索推理 cpWER 最高 **−64.2**；模型自发门控的 F1 达 **0.75–0.80**，全面反超外置信号检测器的 0.53；只训二元门控还附带提升了独立的听觉推理基准（**+10pp**），且零灾难遗忘。
- **门控塌缩攻坚**：用监督微调「破饱和」彻底矫正训练诱导的门控塌缩，三模型干净流门控准确率从 **0 恢复到 1.0**，转写不退化反而普遍改善。

**贯穿全链的五条铁律**：①增益集中在模型真正失败处；②训练内化是主增益；③越弱的模型收益越大；④前置能力与上下文设计精准度决定成败；⑤能力可内化、可迁移。全程零泄漏，所有增益是真实理解而非抄答案。

# 一、问题：从「加了上下文却在抄答案」说起

## 1.1 背景与动机

Omni 模型（能同时处理音频、文本等多模态的大模型）近年在语音理解上进步飞速，但它们的强项集中在「干净、单一说话人」的录音上。一旦录音变「脏」——两个人同时说话造成声音重叠、背景有持续的环境噪声、说话人离麦克风很远产生混响——模型对「谁在什么时候说了什么」的判断就会急剧退化：要么把两个人的话混成一段、要么漏掉一个人、要么把噪声当成语音。

这类「复杂声学场景」恰恰是真实世界里最常见的：会议记录、客服通话、车载语音、多人访谈。一个自然的工程直觉是：既然模型自己听不清，那就给它一份「参考资料」（context，上下文），把场景里的有用信息先告诉它，帮它理解。本项目就从这个直觉出发，但很快撞上了一个致命的陷阱。

## 1.2 致命陷阱：加了上下文，分数飙高，却是「抄」来的

我们做的第一版（记为 V1，最朴素的实现）把上下文做成了一份「完整的 XML 结构化标注」——里面包含了每个说话人说了什么的转写文本。结果惊人：三个模型加上这份上下文后，平均分从约 50% 直接飙到约 95%。

| 模型 | 无上下文 | 加「含答案」上下文 |
|---|---|---|
| Qwen3-Omni 30B | 65.6 | 93.3 |
| MiniCPM-o 8B | 53.4 | 96.8 |
| Ming-flash 104B | 40.1 | 91.1 |

这个数字好得不真实。我们没有立刻庆祝，而是设计了三项诊断去验证「模型到底是听懂了，还是在抄上下文里的答案」。

## 1.3 三项诊断：坐实「抄答案」

- **诊断一·答案泄漏分析**：在一个 600 条的子集上逐任务统计「上下文里直接含有答案的比例」（可抄率）。结果转写类、说话人归属类任务的可抄率高达 **~100%**——也就是说，模型根本不需要听音频，照抄上下文就能答对。

- **诊断二·污染测试（盲抄率 BCR）**：我们故意往上下文里注入**错误**的标注，看模型会不会跟着错。如果模型是真听懂了，给它错答案它应该坚持自己听到的；如果它在抄，给错答案它就会抄错。结果 Qwen3/MiniCPM 的「盲抄率」高达 **94%–99.8%**——给错答案几乎全跟着错，铁证在抄。

- **诊断三·静音探针（决定性）**：这是最干脆的测谎实验——把音频整个换成**纯静音**，只保留含答案的上下文。如果模型还能答对，那它一定是在抄（因为根本没有音频可听）。结果含答案上下文下模型在静音输入上仍然答对，确凿坐实了泄漏。

静音探针后来成为整个项目贯穿始终的「测谎仪」：任何号称「上下文带来的提升」，都必须通过静音探针（去掉音频就答不对）才算数。

## 1.4 核心命题确立

三项诊断把问题彻底厘清了：**分数高 ≠ 真听懂**。整个项目要破解的核心问题随之确立——

> 如何设计上下文，使模型的提升确凿来自「听懂音频」而非「抄答案 / 标签泄漏」？并且把这种上下文做成可泛化、可开源、可自动生成的数据合成流水线，而非依赖人工标注？

这个命题贯穿后续全部五个阶段。每一处「提升」都要回答两个问题：它在模型真正失败的地方有效吗（而不只是在简单样本上锦上添花）？它经得起静音探针的检验吗（是听懂还是抄）？

# 二、方法：AGSC + 三道保险 + 门控 + 零泄漏合成管线

## 2.1 五条方法学原则

在动手设计上下文之前，我们先从 V1 的失败里提炼出五条贯穿全程的方法学原则：

1. **对症失败模式**：不同的「脏」对应不同的失败模式，线索要对症——重叠场景给「该听谁」的说话人时间窗；噪声场景给「有多吵、哪里有语音、可能涉及哪些词」的噪声档位与候选词。
2. **零泄漏且可证**：线索绝不能含可照抄的答案，且这一点要能被程序化地证明（泄漏门禁）和实验性地验证（静音探针）。
3. **强前置决定线索质量**：线索是由前置工具（说话人日志、ASR、语音分离）自动算出的；前置工具不够强→算出错线索→误差累积反伤模型。前置组件必须实测选优，不能想当然。
4. **难度切片看真相**：复杂场景的样本难度差异极大，全样本均值会把「难样本上的真实增益」和「简单样本上的噪声」平均掉。必须按 SNR × 重叠程度 × 难易切片，重点观察模型真正失败的难样本。
5. **从外部上下文到内在能力**：先在推理时证明线索有用，再用训练把它内化成模型自身能力，并在独立的下游 benchmark 上验证这种能力的外部效度。

## 2.2 AGSC：只给线索，不给答案

基于这些原则，我们设计了 **AGSC**（Audio-Grounded Scaffold Context，音频接地的脚手架式上下文）。它的核心比喻是：帮一个朋友在嘈杂的派对里听清某人说话——

- ❌ 错误做法（抄答案）：直接告诉他「那个人说的是『明天三点开会』」——他根本不用听了。
- ✅ AGSC 做法（给线索）：只告诉他「你要听的那个人在第 3 到 6 秒说话」或者「背景很吵，内容可能涉及这几个词」——具体说了什么，他还得自己竖起耳朵去听。

AGSC 针对三类场景对症给出不同的脚手架式线索：

- **重叠场景**：给每个说话人一个**时间窗**（例如「目标说话人在 0.00–3.85 秒说话，性别 = female」），但绝不含他所说的文字内容。模型必须用这个窗去音频里定位、然后真听才能转写出来。
- **S1 单人 + 噪声**：给**噪声档位**（吵 / 中 / 静，由 SNR 估计得出）+ **语音活动区**（VAD 给出哪些时间段有人在说话）+ **部分打乱的候选词**（把真词和干扰词混在一起、只保留约三分之二并打乱顺序；数字这类「一抄就对」的原子答案直接不给）。
- **S2 重叠 + 噪声（最难）**：先用语音分离器把两个人的声音拆开，对每个人各自跑 ASR 出一份「草稿」，然后**只给每个人草稿里的「部分打乱关键词」**——不给整句，因为整句草稿即使有错也会被模型当答案强行照抄。

## 2.3 三道非泄漏保险

AGSC 的「零泄漏」不是靠承诺，而是靠三道可验证的保险共同守住：

1. **防抄契约（Audio-First Contract）**：在提示里显式声明——这些线索是自动工具产生的、未经验证、可能含有错误和干扰项、不含完整答案、一切以音频为准、不要照抄线索。这一段文字本身还有意外收获：它对强模型有「专注聆听」的行为引导作用（见第五章流式实验的发现）。
2. **泄漏门禁**：一道程序化的校验关卡——归一化之后，完整答案绝不允许作为线索的连续子串出现；候选词必须混入干扰项、打乱顺序、只保留一部分。任何违反的样本在生成阶段就被拦下。
3. **静音探针（Silent Solvability）**：把音频换成静音，如果仅凭线索模型还能答对，就判定为泄漏。真实增益要求静音分数足够低——这是事后的实验性兜底，与事前的泄漏门禁形成双保险。

## 2.4 门控：被实证逼出来的关键设计

一个重要的实证发现是：**线索不是越多越好，在不需要它的地方注入会反伤**。在干净、无重叠、高 SNR 的样本上强行注入线索，反而引入噪声、拉低成绩。

于是我们引入「门控」（Gating）：只有当前置工具检出重叠 / 多说话人 / 低 SNR 时才注入线索，否则退回纯音频。门控信号直接来自合成管线（重叠检测、SNR 估计），可以直接部署。

门控的价值有硬数据支撑：**它把整体增益从「无脑全注入」的 +1.2 提升到了 +3.8——约 3 倍**。门控也因此成为后续流式场景和强化学习阶段反复出现的核心组件，乃至最后「门控塌缩」攻坚的主角。

## 2.5 零泄漏自动合成管线（可泛化、可开源）

AGSC 线索由一条六阶段的自动管线从任意音频生成，不需要人工标注：

```
任意音频
 → [1] 说话人日志 (pyannote-3.1)           谁 / 何时说话
 → [2] 分段转写 (Mega-ASR)                 每段大概说了啥（仅作候选）
 → [3] 重叠检测 / VAD 语音区 / SNR 档位       噪声场景新增
 → [4] 重叠+噪声: SepFormer 分离两人 → 各自 ASR → 部分打乱关键词
 → [5] 门控: 仅"重叠/低SNR"注入, 干净不注入
 → [6] 零泄漏 predicted-AGSC 导出 (+置信标注)
```

管线的每一个组件都经过实测选型，奉行「宁缺毋滥」——一个不够可靠的组件宁可弃用，也不让它产生错线索去反伤模型：

| 组件 | 选用 | 实测指标 | 选型依据 |
|---|---|---|---|
| 说话人日志 | pyannote-3.1 | DER 0.22 / 说话人数命中 75% | 完胜 CAM++（DER 0.47 / 命中 0%）|
| 分段转写 | Mega-ASR | 噪声 −5dB 下 CER 0.05 | 抗噪 SOTA |
| 语音活动检测 | silero-vad | 稳定定位语音段 | 采用 |
| SNR 估计 | 基于 VAD 的能量比 | 点估 MAE ~9.9dB，但粗档位命中率 79% | 只用粗档，不用点估 |
| 声学场景标签 | AST(AudioSet) | 场景精度仅 29% | **弃用**（实证否决的组件）|
| 语音分离 | SepFormer-whamr | 分离草稿 cpWER 16.8% | 见第四章选型对比 |

**链路泛化实测**：在 12 条真实样本上端到端对比金标，整条管线达到 DER 0.15、说话人数命中 83%、分段 CER 0.20；置信标注分布为 high 9 / medium 3 / low 0，且难例（重叠极重、单人被误切成两人）被置信规则正确地降级为 medium——管线知道自己什么时候不靠谱。

## 2.6 三模型与训练

为了验证规律的普适性，我们选了三个**架构、规模都不同**的 Omni 模型：Qwen3-Omni 30B（中等规模、较强）、MiniCPM-o 8B（小、较弱）、Ming-flash 104B-MoE（最大、零样本最弱）。

训练采用 LoRA 轻量微调，只在注意力投影上注入低秩适配器，可训练参数约占全模型的 **0.05%**。三个模型的接口差异巨大，分别做了适配：

- **Qwen3-Omni 30B**：对其内部真正负责语言的 thinker 模块注入 LoRA，用 DeepSpeed ZeRO-2 四卡数据并行训练。
- **MiniCPM-o 8B**：复刻它 `model.chat` 的 teacher-forcing 输入构造，让 `forward(data, labels=)` 透传到底层 LLM 出 loss，单卡训练。
- **Ming-flash 104B-MoE（BailingMM2）**：复用其适配器加载（device_map 四卡 naive 模型并行），手动构造输入直接 forward——这一过程攻克了 7 个自定义接口的坑（模板只接受 HUMAN 轮、PEFT task_type 适配、whisper 音频 dtype、cache 的 bool 处理、fused 交叉熵的 inplace_backward 等）。

统一的训练目标是：在门控 AGSC 数据 + 金标数据上，学会「用线索在重叠 / 噪声中归属说话人并转写，且在无线索时也能转写」。

# 三、链路一 · 重叠场景：从泛化到内化

多说话人重叠是 who-said-what 任务的核心难点：两个人的声音在时间上交叠，模型既要分清「这是谁的声音」，又要听清「他说了什么」。重叠场景是整个 AGSC 范式的起点，分三个阶段推进——先验证管线能泛化（Stage A），再验证推理时线索有用（Stage B），最后用训练把它内化（Stage C）。

## 3.1 Stage A：合成管线的链路泛化

在投入大规模实验前，先要确认这条自动管线在真实音频上能不能工作。在 12 条真实样本上端到端对比金标，管线达到 DER 0.15、说话人数命中 83%、分段 CER 0.20，置信标注 high 9 / medium 3 / low 0。关键在于难例被正确降级——当重叠极重、或一个人被误切成两人时，置信规则会把这些样本标为 medium 而非 high，等于管线对自己的不确定性有自知之明。这为后续「按置信度调节线索强度」留了接口。

## 3.2 Stage B：推理时下游验证（训练前，零泄漏）

不训练，只在推理时注入 predicted-AGSC 线索，看增益（Δ 越高越好）：

| 模型 | SparseLibriMix2 Δ全部 | 难1/3 Δ | AMI 真实会议 Δ全部 | AMI 难1/3 Δ |
|---|---|---|---|---|
| Qwen3 | +1.3 | +8.3 | +12.0 | +21.8 |
| MiniCPM | +15.8 | +36.1 | +6.0 | +2.9 |
| Ming | +0.8 | +12.8 | −1.7 | +3.8 |

读这张表的关键是看「难1/3」那两列：6 个高难（模型 × 任务）组合的难样本增益**全部为正**，区间 +2.9 到 +36.1。也就是说，在模型真正失败的地方，线索一致地有效。

这里有一个堪称「铁证」的机制观察：在真实会议数据 AMI 上，模型的 baseline 行为是「过度转写」——把整段会议里所有人说的话都一股脑转写出来（因为它分不清该听谁）。给了 AGSC 的目标说话人时间窗之后，模型聚焦到了目标，输出从平均 **52 个词缩到 8 个词**。由于线索里只有时间窗、零转写内容，这个从「全转」到「精准转目标」的改善，只可能来自「模型用时间窗去音频里定位、然后真听」——这是听懂，不是抄。

## 3.3 Stage C：训练内化（held-out 同集前后对比，cpWER↓）

把 AGSC 数据拿去做 LoRA 微调，然后在同一个 held-out 集上对比训练前后：

| 模型 | SparseLibriMix2 +AGSC 前→后 | AMI 会议 +AGSC 前→后 | 无线索 baseline 前→后 |
|---|---|---|---|
| Qwen3 | 25.3 → **9.0** | 69.8 → **53.3** | 26.6 → 8.4（−16.3）|
| MiniCPM | 48.3 → **13.8** | —（未训 AMI 双音频）| 64.1 → 14.9（−34.5）|
| Ming | 67.4 → **7.0** | — | 68.3 → 7.9（−60.4）|

最右一列「无线索 baseline」是内化的铁证：训练之后，**即使推理时一个字的线索都不给**，模型的成绩也大幅变好（Qwen3 −16.3、MiniCPM −34.5、Ming −60.4）。这说明「会用线索在重叠中听清」这件事，已经被训练成了模型自身的能力，而不再依赖运行时喂线索。越弱的模型降幅越大（Ming > MiniCPM > Qwen3），三者最终收敛到接近的水平（~7–14）——这条「越弱收益越大」的规律，会在后面每一个阶段反复出现。

# 四、链路二 · 噪声场景：前置组件 + 上下文设计决定成败

把环境噪声叠加到多说话人重叠之上，构成本项目最难的场景（S2）。这一章给出整个项目最重要的方法论结论之一：**线索的「产法」和「写法」是决定性的——同一个场景、同一个模型，只换线索怎么生成、怎么措辞，结果可以从「灾难性反伤」翻转到「难样本转正」。**

## 4.1 同场景三种线索设计：天壤之别

![S2 三种 Context 设计对比（Qwen3）：弱前置误导、强前置坏设计灾难、强前置好设计难样本转正](results/fig_s2_design_comparison.png)

| 设计 | 全部 Δ | 难1/3 Δ | 机制说明 |
|---|---|---|---|
| 弱前置：带噪 pyannote 光秃时间窗 | −1.6 | −2.0 | 噪声让说话人日志退化，时间窗不准 → 误导模型 |
| 强前置但坏设计：降噪 + ASR 整句草稿 | **−11.7** | −0.8 | 重叠转错的「流畅错草稿」被模型当答案强锚定 → 误差爆炸 |
| 强前置 + 好设计：SepFormer 分离 + 门控关键词 | −1.8 | **+1.3** | 既有用又不可照抄，避免灾难、难样本转正、零泄漏 |

三种设计的对比讲了一个深刻的道理：

- **第一种（弱前置）**的问题在「产法」：用一个被噪声搞退化了的说话人日志去产时间窗，窗本身就是错的，错线索自然误导模型。
- **第二种（强前置但坏设计）**最值得警惕：它用了强降噪和 ASR，产出的是一份**流畅但有错**的整句草稿。流畅恰恰是它的毒性所在——模型会把这份读起来很顺的草稿当成答案强行锚定照抄，结果错误被放大到 **−11.7** 的灾难级别。这印证了一个反直觉的点：线索质量不够时，**越「像答案」的线索越危险**。
- **第三种（强前置 + 好设计）**才是正解：用专门的语音分离器（SepFormer）把两人分开，各自出草稿（草稿 cpWER 约 27%，适度不准），然后只取「部分打乱的关键词」。这样的线索既保留了有用信息，又因为打乱和残缺而无法被照抄——难样本上转正到 +1.3，零泄漏。

**结论**：「前置组件能力 + 上下文设计精准度是决定性的」——线索质量不够时不是没用，是**反伤**。

## 4.2 前置分离器选型：SepFormer vs SAM Audio（实测史诗）

既然分离器这么关键，到底用哪个？我们做了一场实测对比。候选之一是 Meta 新出的 SAM Audio（Segment Anything for Audio，「按描述分离任意声音」的通用分离器）。

在同 12 个 S2 样本上，比「分离 → ASR 草稿」的 cpWER（越低 = 分离越准）：

| 前置 | 草稿 cpWER | 结论 |
|---|---|---|
| **SepFormer-whamr** | **16.8%** | 采用（专为「2 人 + WHAM 噪声」训练）|
| SAM Audio base | 43.5% | 弃用 |
| SAM Audio small-tv | 55.8% | 弃用 |

结论很清楚：SAM Audio 是个「通才」——按文字描述提取任意声音事件（狗叫、关门、某种乐器）很强，但**两个同为人声的说话人，它分不干净**（即便给时间窗 span 提示也不行）。who-said-what 这种「同类人声分离」用专才 SepFormer 显著更优。这是「实证选型、宁缺毋滥」原则的典型案例：不因为 SAM Audio 名气大就用它，而是用分离草稿的 cpWER 数字说话。

附带一段诚实的工程记录：为了在老驱动（driver 470，只支持 CUDA ≤11.4）上跑通 SAM Audio，我们做了约 30 余处依赖与兼容修复（torch 2.5.1+cu118、给 torchcodec 做 soundfile 打桩、关闭 cuDNN 与 torch.compile 走 eager、stub 掉 ImageBind/CLAP reranker）。最终撞上一道硬墙：SAM Audio 的音频解码层 torchcodec 强依赖 CUDA 12/13 的 libnvrtc.so.13，老驱动无法满足。我们把这道墙如实记录下来——它和后面 DeepSpeed 遇到的 CUDA-12 算子墙同源，都是老硬件的物理边界。

## 4.3 三模型 N-C 推理时结果（SepFormer 门控前置）

用选定的最优前置（SepFormer + 门控关键词），三模型推理时的增益：

| 模型 | S1 全部Δ | S2 全部Δ | S2 低SNR Δ | S2 难1/3 Δ |
|---|---|---|---|---|
| Qwen3（S2 base 24.7）| +0.2 | −1.8 | −1.5 | **+1.3** |
| MiniCPM（S2 base 65.8）| +0.1 | **+3.2** | **+8.9** | **+15.7** |
| Ming（S2 base 71.1）| +0.0 | −1.0 | −1.5 | **+18.8** |

S2 难1/3 三模型全正（+1.3 / +15.7 / +18.8）。但注意：全样本的净增益有正有负——这是因为在简单样本上，不完美的线索成了噪声，把难样本上的正增益在均值里抵消掉了。对有空间的弱模型 MiniCPM，整体净增益为正（+3.2，低 SNR 段 +8.9）；对强模型 Qwen3，简单样本的噪声让全样本净增益微负。**这种「推理时净增益不稳」正是需要训练内化的原因**——把能力训进模型，就不再受运行时线索质量波动的影响。

## 4.4 Stage N-D 训练内化（held-out 前后，cpWER↓）—— 决定性结果

![训练内化：三模型 S2 重叠+噪声 cpWER 训练前后（左=全部样本，右=最难1/3）；灰=训练前基线，绿=训练后同样不给线索](results/fig_training_internalization.png)

这是整个项目最重要的一张图。灰柱是训练前不给线索的基线，绿柱是用 AGSC 数据 LoRA 微调后**同样不给线索**的成绩：

| 模型 | baseline 前→后 | +AGSC 前→后 | 难1/3 前→后 |
|---|---|---|---|
| Qwen3-Omni | 26.4 → **8.7**（−17.7）| 28.6 → **8.6**（−20.0）| 50.5 → **14.6**（−35.9）|
| MiniCPM-o | 65.8 → **14.0**（−51.8）| 62.6 → **22.5** | 70.6 → **18.1**（−52.4）|
| Ming-flash | 71.1 → **15.0**（−56.1）| 72.1 → **14.7**（−57.4）| 77.6 → **15.4**（−62.2）|

四个层层递进的解读：

1. **能力做实**：三模型的 S2 cpWER 从 26–71 全部降到 9–15，最难 1/3 样本从原来基本听不清（50–78）降到 15–18。这是抄不来的实质能力跃升。

4. **静音探针守住非泄漏**：全程线索零转写、静音探针验证，这些增益是真听懂。

## 4.5 S1 单人 + 噪声：诚实记录的「无空间」

不是所有场景都有增益空间。S1（单人 + 噪声，无重叠）上，三模型本就近天花板（中文噪声 ASR 的 CER 仅 3–4%），训练后基本持平（±0.3）。这不是方法失效，而是数据特性——任务对这些模型来说不够难，没有可改善的空间。我们把它如实记录，并指出：想在 S1 上看到增益，需要更难的噪声集（更低 SNR、远场、非中文）。

# 五、链路三 · 流式场景：增量增益曲线 + 实时门控

前两章是 offline 设定（模型一次性拿到完整音频和完整线索）。但真实部署往往是流式的：音频一秒一秒地来，t 时刻模型只能听到前 t 秒，线索也只能由前 t 秒算出。这一章回答两个部署关键问题：**线索要积累几秒才有用？能否实时门控、只在复杂段注入？**

实验规模：55 条 ≥8 秒的样本 × 7 个前缀长度 × 3 个条件 × 3 个模型（共 5,235 次推理）+ 30 条合成流 × 5 种门控策略 + 360 个一 token 探针窗。这一章用的是**基座模型**（未加训练 LoRA），专门考察推理时的增益规律。

## 5.1 实验设计的巧思：两条曲线分离两个混杂因素

流式增益其实被两个因素纠缠：①模型只听到截断音频，听不到尾部，本身就转不全；②线索由短音频算出，还没「长出来」。要分清是哪个在起作用，我们设计了两条曲线：

- **曲线 A（真流式）**：模型听截断音频 + 前缀线索——这是真实流式，但两个因素纠缠在一起。
- **曲线 B（线索成熟度）**：模型听**完整音频**，但线索只由前缀算——单独隔离出「线索要积累几秒才有用」这一个问题，排除「截断音频听不到尾部」对分数的稀释。

这个设计让我们能干净地回答「线索成熟需要几秒」，而不被「音频没听全」干扰。

## 5.2 线索需 3–6 秒「成熟」，增益集中在失败点

线索的信息量随音频积累而增长（线索由 SepFormer 分离 + ASR 算出，音频越长能算出的关键词越多）：

| 前缀 | 1s | 2s | 3s | 4s | 6s | 8s | full |
|---|---|---|---|---|---|---|---|
| 平均关键词数 | 0.0 | 2.0 | 5.0 | 7.2 | 11.8 | 15.1 | 16.9 |

**曲线 A（真流式）难1/3 增益**（GCG，pp，正 = 有益）：

![难1/3 切片曲线A：绿色阴影=流式线索在失败点的增益区](results/stream_hard_curve.png)

| 模型 难1/3 | 1s | 2s | 3s | 4s | 6s | 8s | full |
|---|---|---|---|---|---|---|---|
| Qwen3 | +0.3 | −0.3 | **+8.2** | +2.3 | −6.6 | +2.1 | +1.6 |
| MiniCPM | −0.2 | +0.9 | +0.2 | −0.8 | +0.1 | **+3.9** | −1.4 |
| Ming | −0.1 | −9.1 | +3.4 | +5.3 | **+8.3** | +6.3 | **+12.3** |

**曲线 B（线索成熟度）难1/3 增益**：

![难样本增益随线索成熟而增长：三模型增益线全程为正 + 线索信息量柱（右轴）](results/stream_hard_gain_info.png)

| 模型 难1/3 | 1s | 2s | 3s | 4s | 6s | 8s | full |
|---|---|---|---|---|---|---|---|
| Qwen3 | +8.8 | +6.2 | +1.2 | +2.3 | +0.2 | −0.1 | +1.9 |
| MiniCPM | +5.3 | +8.6 | +5.7 | +2.8 | +10.3 | **+15.7** | +6.5 |
| Ming | +7.6 | **+15.1** | +14.8 | +13.0 | +16.7 | +14.1 | **+21.2** |

三段式读法：

- **1–2 秒：无增益甚至负增益**。1 秒音频算出 0 个关键词、2 秒约 2 个——线索物理上还没有信息可给。Ming 在真流式 1 秒处甚至被「有框架没内容」的空提示拖累到 −11.2。这说明「超短音频就能显著提升」是不成立的，但这是信息量的物理边界，不是方法失效。
- **3–4 秒：增益窗口打开**。Qwen3 在 3 秒处难1/3 增益 +8.2（此刻它只听到 3 秒、自身 cpWER 高达 78，几个关键词就能显著救场）；Ming 难样本转正。
- **6 秒之后**：强模型自己能听清了，线索增益回落甚至为负；弱模型 Ming 的难样本增益持续走高（+8.3 → +12.3）。

最关键的读数在曲线 B 的弱模型上：**Ming 用 2 秒音频算出的线索就达到 +15.1，6 秒拿到 +16.7，已是全量增益（+21.2）的 79%**。这就是「流式线索何时够用」的答案——**不必等 offline 全量音频，给线索 3–6 秒的生长期即可**。成熟之后，流式增益与 offline 同向、同规律（集中在弱模型 + 难样本）。

## 5.3 实时门控：省 65% 注入量，但「一句话措辞」决定成败

![门控五策略：上排 cpWER（黑框=最优），下排干净段召回（虚线=基线水平）](results/fig_gate_strategies.png)

用 30 条 [干净 4s | 复杂 4s | 干净 3s] 的合成流测实时门控（拼接构造让复杂区真值位置免标注）。

**外置信号检测器**（逐 1 秒窗 pyannote 重叠检测 + 滞回状态机平滑）：逐秒 F1 0.53，触发延迟 1.72 秒，干净段豁免率 75%，注入量比全程注入省 65.1%。F1 0.53 够用但不优秀。

**端到端五策略（Qwen3 面板）**——这里发现并修复了一个重要现象「注意力虹吸」：

| 策略 | cpWER↓ | 干净段召回 | 复杂段召回 | 耗时s |
|---|---|---|---|---|
| baseline | 38.1 | 97.6 | 40.3 | 8.06 |
| always 盲注入 | 37.9 | 98.2 | 38.7 | 7.86 |
| gated（检测器 + 时间窗）| 48.2 | 73.5 ⚠ | 37.2 | 6.83 |
| gated_gt（真值窗）| 41.7 | 85.6 ⚠ | 38.1 | 7.47 |
| **gated_v2（真值窗 + 「仍需完整转写全程」）** | **36.2 ✅** | **97.6 ✅** | 39.5 | 8.26 |

**注意力虹吸**：朴素的「在 4–8 秒检测到复杂段」时间窗线索，会把模型的注意力整个吸进那个窗口，导致窗口外的干净段被静默漏转——Qwen3 的干净段召回从 97.6 暴跌到 73.5，MiniCPM 最惨跌到 57.3。模型变成了「只顾复杂段的话痨」。

**一句话修复**：在线索里加一句「仍需完整转写全程」（gated_v2），Qwen3 的干净段召回弹回 97.6，整体 cpWER 还做到了五策略最优 36.2（低于基线 38.1）。**一句措辞 = 好几个点的 cpWER**——这是「上下文设计精准度决定成败」在流式场景的再次印证，也成为后面流式规律里新增的第五条铁律。

两点补充：①门控比全程注入省 65% 注入量、推理还更快（线索更短）；②检测器质量是门控上限——gated（F1 0.53）和 gated_gt（真值窗）之间 Qwen3 差 6.5pp、MiniCPM 差 5.8pp，所以把流式重叠检测的 F1 提上去是性价比最高的改进点。门控收益还是模型相关的：gated_v2 对 MiniCPM 有副作用（话变啰嗦、插入错误增多），说明没有万能 prompt。

## 5.4 一 token 探针：模型自己当门控开关（零样本）

能不能让模型自己判断「这段要不要线索」，而不依赖外置检测器？我们让模型对每个 1s/2s 窗只输出一个词 COMPLEX 或 CLEAN：

| 模型 | 窗长 | 总准确率 | CLEAN 准确 | COMPLEX 准确 | 单窗延迟 |
|---|---|---|---|---|---|
| Qwen3-30B | 1s / 2s | 67% / 73% | 100% / 90% | 33% / 57% | 1.9s / 2.0s |
| MiniCPM-8B | 1s / 2s | 72% / 68% | 100% / 80% | 43% / 57% | **0.30s** |
| Ming-104B | 1s / 2s | 50% / 58% | 100% / 97% | 0% / 20% | 1.0s |

三个观察：①干净识别近满分，但复杂识别系统性偏低——模型「听得出有点不对，但不敢喊复杂」，决策边界保守，Ming 干脆全报 CLEAN；②窗口从 1s 加到 2s，复杂识别普遍上涨（33→57、43→57、0→20），说明 2 秒是更合理的最小决策单元，与「线索 3 秒起效」互相印证；③MiniCPM 的 0.30 秒单窗延迟证明「用小模型当流式哨兵 + 大模型干活」的架构实时可行。

这个「表征里有信号、但边界保守、零样本不敢报警」的现象，恰恰是强化学习能调的对象——它直接成为下一章 GDPO 门控内化的起点。

## 5.5 与 offline 结论的对照 + 部署蓝图

offline 的四条规律在流式设定下全部复现，并新增一条：

| offline 规律 | 流式验证 |
|---|---|
| 增益集中在模型失败点 | ✅ 曲线 B 难1/3 全面正、全样本贴零 |
| 前置能力 + 设计决定成败 | ✅ 短前缀线索空 → 无增益；注意力虹吸 → 一句话翻盘 |
| 越弱的模型收益越大 | ✅ 成熟度难1/3：Ming +21.2 > MiniCPM +15.7 > Qwen3 |
| 干净场景注入有害 → 需门控 | ✅ 盲注入 / 坏门控伤干净段，门控 + 精准措辞修复 |
| **（新）线索措辞一句话改变门控成败** | gated → gated_v2：cpWER 48.2→36.2、召回 73.5→97.6 |

由此得出的流式部署蓝图：音频流先经一个轻量哨兵（现阶段是 F1 0.53 的信号检测器，目标是 GDPO 后模型自发门控）做 2 秒窗复杂度检测；干净段直接转写（零额外延迟、豁免率 75%）；复杂段等线索成熟（≥3 秒积累）后注入，措辞必须含「仍需完整转写」；长期则用 AGSC 数据训练内化，线上直接用 baseline prompt 部署，线索仅作弱模型 / 极难段的运行时补充。

# 六、链路四 · 强化学习 GDPO：三阶段全链路内化

到第五章为止，AGSC 已经在重叠、噪声、流式三大场景被证明有效，并通过监督微调把「转写能力」内化了。但还有三块能力停留在「外置链路」，没有进入模型内部：

1. **门控决策**——「这段要不要线索」目前靠一个外置信号检测器（F1 仅 0.53、延迟 1.72 秒、占独立 GPU），而模型零样本「不敢报警」（COMPLEX 识别最低仅 20%）；
2. **不同成熟度流式线索的「用而不盲信」**；
3. **输出契约与声学场景元认知**。

这一章用强化学习把这三块能力一次性内化进模型。骨架是 **GDPO**（Group reward-Decoupled Normalization Policy Optimization，组奖励解耦归一化策略优化，arXiv:2601.05242）。整个强化学习分三个阶段：阶段一做门控单点 RL 的方法选型，阶段二做全链路内化，阶段三修复多目标张力。

## 6.1 为什么是强化学习，为什么是 GDPO

**为什么用 RL 而非继续 SFT**：门控正确性、转写质量、输出格式，这些都是不可微的序列级信号；而且「什么是好门控」依赖下游后果——一次注入到底有没有真帮到转写，得看最终结果才知道。这天然是一个策略优化问题。SFT 只能模仿「正确答案」，无法表达「这个选择比那个选择相对更好」，而 RL 的相对优势机制正是为此而生。

**为什么用 GDPO 而非更常见的 GRPO**：本任务的奖励天然是多元的——格式 + 门控 + 转写三个奖励同时存在。GRPO 的做法是「把多个奖励先加总成一个标量，再做组内归一化」，这会把不同的奖励组合压成相同的优势值（信号塌缩），实测训练约 400 步后就崩坏。GDPO 用三个机制解决这个问题：

- **Eq.4 解耦归一化**：每个奖励分量**单独**做组内 z-score 标准化，而不是先加总；
- **Eq.7 解耦权重**：标准化后再加权求和（本项目用 w=(1,2,2)），权重可调地控制各目标的相对重要性；
- **Eq.6 batch 级归一化**：在 batch 层面再归一，稳定训练。
- 另有 **Eq.8 条件奖励**：r_k × 𝟙[r_l ≥ t]，即只有当某个「难奖励」达标时才发放「易奖励」，防止模型刷容易拿的分而忽略难的目标。

本项目把这三个（加条件奖励共四个）机制全部用上，并各自拿到了实测验证（阶段一的选型对照证明解耦归一化、阶段三的张力修复证明解耦权重 + 条件奖励）。

## 6.2 关键技术机制：解析策略 + ε-探索

强化学习有一个常见的退化陷阱：当模型对某个判断过于自信时，一组（G 个）采样会全部一致 → 组内零方差 → 优势恒为 0 → 梯度恒为 0，训练停滞。这是 GRPO 和 GDPO 共同的退化点。

我们用两个机制解决（这也是本项目的方法创新）：

- **解析策略**：把「2 秒窗 → 二元判定 COMPLEX/CLEAN」的最小门控任务，建模成 π(a) = softmax(两个动作各自 teacher-forcing 序列的 logprob)——精确、每窗只需 2 次前向（比文本 rollout 快约 10 倍）。
- **ε-探索行为策略**：用 μ = (1−ε)·π + ε·uniform（ε=0.25）做行为策略采样，再用 PPO 比率 s = π_θ(a)/μ(a) 修正偏差（忠实于 Eq.3 的 clip 目标）。这样即使策略已经很自信，探索仍能注入多样性，避免组内零方差。

这套机制还有一个意外的工程价值：当 MiniCPM 的 chat 采样接口失效时（见后文工程坑），解析策略 + ε-探索可以直接替代文本 rollout，绕过采样问题。

## 6.3 三奖励设计

- **阶段一（门控单点）双奖励**：r_gate（判定正确性）+ r_down（实测查表的下游收益）。后者是本项目的巧思——每条训练流离线预先测出「注入 vs 不注入」的真实 cpWER 差 Δ_s，复杂窗判 COMPLEX 就给 tanh(Δ_s/0.2)，干净窗误判给 −0.5。这样 RL 循环内不需要额外做转写推理，奖励却直接反映了真实下游后果。
- **阶段二（全链路）三奖励**：r_format（输出契约）+ r_gate（场景真值，合成流免标注）+ r_asr = 1−cpWER（每个 rollout 实算多参考置换不变 cpWER 的真奖励，参考 = 干净段 Mega-ASR 伪标注 + 重叠段金标）。

## 6.5 阶段二：全链路 Gate-then-Transcribe 内化

阶段一只训了门控这一个点。阶段二要把门控、转写、流式线索利用三件事**统一进一次 GDPO 训练**。

**统一回合设计**：每个训练回合 = 一条流音频（70% 是 [干净 | 重叠+噪声 | 干净] 复合流，30% 是纯干净流）+ 一份随机成熟度的线索（none / clue@2s / clue@4s / full）。模型**单次生成**：第一行输出 `GATE: COMPLEX|CLEAN`（相当于自发的门控开关），随后逐说话人输出转写。一次前向同时训练门控、转写、线索利用三种能力。这种「成熟度随机化」相当于给模型上了一门「流式线索分批到货」的全谱系利用课程——弱则靠听、强则善用、干净不被带偏。

只用了 **160 条合成流、LoRA、1–2 个 epoch**。

# 七、链路五 · 门控塌缩攻坚：SFT 破饱和

强化学习把整条链路内化了，但大规模训练带来一个意料之外的系统性副作用。这一章是一段完整的「诊断 → 证伪 → 再诊断 → 破解」攻坚叙事，也是整个项目方法学价值最集中的部分。

## 7.1 现象：训练后门控全部「卡死常开」

三个模型在大规模训练之后，干净流的门控**全部塌缩成了常开**——连干净的单人录音也判成「复杂」、坚持要注入线索。干净流门控准确率（应该输出 CLEAN）：Qwen3 和 MiniCPM 掉到 0，Ming 只剩 0.11。

根因不难定位：训练分布里干净样本（clean episode）占比太低，模型学「懒」了——反正全判「复杂」也不怎么扣分，于是收敛到「恒判 COMPLEX」这个退化解。问题是怎么把它救回来。整个攻坚完全由机制诊断驱动，而不是盲目调参。

## 7.2 第一条路（B 软门控）：证伪，但失败得有价值

第一个想法借鉴了 IRAF 那篇工作的「连续概率门控」思路：模型在输出 COMPLEX/CLEAN 这个词之前，其实有一个连续的置信概率 π。也许 argmax（最终选哪个词）塌缩了，但底层的连续概率 π 还保留着区分度？如果是这样，就可以按 π 分三档注入（≥0.8 给全线索 / 0.5–0.8 给弱线索 / <0.5 不注入），软性地救回门控。

为了验证，我们写了一个 forced-choice 的 π 探针——在 "GATE:" 之后直接比较 COMPLEX 和 CLEAN 两个词首 token 的 softmax 概率，单次前向就能精确读出 π。结果是一盆冷水：

- **训练后的模型**：干净流 π 中位数 = 1.000，复杂流 π 中位数 = 1.000——两者**完全重叠**，毫无区分度。
- **未训练的 base 模型**（方法学对照）：干净流 π = 0.012，复杂流 π = 0.915——区分得清清楚楚。

base 对照证明了探针本身是有效的（它能测出区分度），所以问题出在训练——**训练把连续概率也压饱和了**，塌缩发生在连续概率层，而不只是 argmax 层。软门控的三档阈值因此退化成了一档（clean 和 mix 都落在 ≥0.8），救不了。

这是一个有价值的负结果：它界定了「连续门控」这条路成立的前提——概率必须有「梯度可达性」，一旦被训练压饱和就失效。

## 7.3 第二条路（A-RL 续训）：也失败，但机制清晰

第二个想法是回到强化学习：加大门控奖励权重、补上干净样本，继续 RL 续训，把门控「掰」回来。我们用 clean 上采样 + w_gate=3 热启动 GDPO 续训了 76 步。结果门控奖励卡在 0.5（瞎猜水平）上不去，而且 76 步里有 18 步梯度直接是 0。

机制层面有两个叠加的原因：

1. **饱和点梯度消失**：COMPLEX 和 CLEAN 两个词的 logit 已经差了约 20，softmax 梯度 ∂L/∂z ∝ p(1−p) 趋近于 0，强化学习的策略梯度信号几乎传不回去；
2. **GDPO 组内优势抵消**：在已饱和的策略下，一组 rollout 几乎全部同向（都输出 COMPLEX），组内奖励无差异，优势 A_i = (r_i − μ)/σ ≈ 0，梯度自然为 0。

结论很硬：**on-policy 强化学习救不动一个已经饱和的门控**——这是 RL 矫正深度饱和策略的固有困难。

## 7.4 第三条路（A-SFT 破饱和）：成功

前两次失败把方向逼得很清楚了。RL 之所以失败，是因为它在饱和点没有梯度。那什么东西在饱和点梯度反而大？——**监督学习的交叉熵**。损失 L = −log p(CLEAN)，当 p(CLEAN) → 0（越饱和）时，L → ∞ 且梯度极大，恰好与 RL 互补，正好能「撬开」饱和。

做法：用干净样本的金标 "GATE: CLEAN" 做 teacher-forcing 监督微调（mix 流则学 "GATE: COMPLEX"），干净样本上采样到一半占比，热启动自已塌缩的 csb LoRA。

- **第一版（v1，只学 GATE 行）踩了个坑**：门控确实治好了（gate_acc 0 → 1.0），但模型把「输出完 GATE 行就结束」也学了进去——target 末尾的停止符让它在判完门控后不再继续转写，转写整个截断了（mix cpWER 飙到 100，召回 0）。
- **第二版（v2，GATE 行 + 完整金标转写）修复**：把 target 改成「GATE 行 + 完整转写」，让模型学会「判完门控继续转写」。两全其美。

SFT 只跑了 90 步，gnorm 稳定在 12–16（对比 RL 续训的约 0，强梯度一目了然），loss 单调收敛到 0.1–0.3。

## 7.5 结果：三模型门控全部治住，转写还更好

![门控塌缩矫正前后：左=干净流 gate_acc 三模型全部 0→1.0，中=复杂段召回提升，右=mix cpWER 不退化](results/fig_gatefix.png)

| 模型 | 干净流 gate_acc | clean cpWER↓ | mix none/t2/full cpWER↓ | 复杂段召回↑ |
|---|---|---|---|---|
| Qwen3 | 0.00 → **1.00** | 6.4→6.2 | 15.8/16.7/16.1→14.6/14.9/15.0 | 80.3→81.5 |
| MiniCPM | 0.00 → **1.00** | 6.6→8.1 | 20.3/21.0/22.6→21.6/18.3/17.9 | 71.3→77.7 |
| Ming | 0.11 → **1.00** | 8.5→7.4 | 19.0/19.0/18.1→17.5/18.3/17.2 | 74.6→80.9 |

三模型的干净流门控全部从塌缩恢复到满分（1.0），而转写不仅没退化，反而普遍改善（监督用的是金标转写，顺带强化了 ASR）。π 探针复测 Qwen3 的干净流 π 从 1.0 拉回到 0.002，区分度恢复到比 base 还干净。

## 7.6 方法学启示

这场攻坚最大的价值不在「治好了门控」这个结果，而在它揭示的方法学：**先诊断问题发生在哪一层（argmax / 连续概率 / 梯度），再决定用什么手段矫正。** 软门控失败是因为问题在连续概率层、概率也饱和了；RL 失败是因为问题在梯度层、饱和点没梯度；SFT 成功是因为交叉熵恰好在饱和点提供大梯度。B 的负结果不是浪费——它精确诊断了根因，并指明了 A 的方向。这套「诊断驱动」的攻坚范式，比任何单一技巧都更可复用。

# 八、实时性：加 Context 几乎不影响速度

一个上下文增强方法要能部署，必须回答「加了线索会不会拖慢推理」。我们在三模型 × 两场景、每组 20 样本上实测了端到端墙钟（含音频处理 + prefill + decode）：

![实时性：三模型×两场景端到端墙钟，baseline vs +AGSC：灰绿柱几乎等高](results/fig_latency_overhead.png)

| 模型 | 场景 | baseline | +AGSC | Δ 相对 | 输入 token | 输出 token |
|---|---|---|---|---|---|---|
| Qwen3 30B | S1 | 3.16 | 3.28 | +3.6% | 129→244 | 11.0→10.8 |
| Qwen3 30B | S2 | 6.54 | 6.50 | −0.6% | 192→347 | 33.2→32.8 |
| MiniCPM 8B | S1 | 0.50 | 0.54 | +9.1% | 92→206 | — |
| MiniCPM 8B | S2 | 0.96 | 0.96 | −0.5% | 146→301 | — |
| Ming 104B | S1 | 1.74 | 1.95 | +11.9% | 172→286 | 9.2→9.2 |
| Ming 104B | S2 | 2.50 | 4.23 | +69% | 270→427 | 14.1→26.7 |

三点解读：

1. **prefill 开销可忽略**：AGSC 只给 prompt 增加约 115（S1）/ 155（S2）个文本 token，而 prefill 的大头是音频 token，所以墙钟差落在 ≤0.21 秒 / ≤12% 的区间（多数 ≤4%），部分场景因测量噪声还略快。
2. **唯一显著的 Ming-S2（+69%）不是「上下文税」，而是「模型把活干完了」**：它的输出 token 从 14 翻倍到 27——在线索帮助下，Ming 终于把两个重叠说话人**都**转写出来了（而 baseline 会塌缩成只转一路、欠转写）。多花的时间是「把活干对」的代价，不是上下文的开销。相对涨幅在小模型上更醒目，只因为它们的基数本就小（0.5–2 秒）。
3. **线索离线预生成，不进在线路径**：S2 线索生成实测每样本 1.56 秒（分离 0.21 + 双路 ASR 1.33），可批量化、可缓存，不占在线推理时间。

**部署最优解**：训练内化之后 baseline ≈ +AGSC，线上可以直接用 baseline prompt 部署——零额外推理延迟，同时保留训练带来的全部精度增益。AGSC 在实时性上没有实质代价。

# 九、贯穿全链的核心规律（五条铁律）

把五个阶段的发现拢到一起，有五条规律贯穿始终、跨场景反复验证：

1. **增益集中在模型真正失败处**。所有场景的难 1/3 样本增益均为正，而简单样本上不完美的线索成了噪声——这既解释了「全样本净增益为何不稳」，也是「为什么必须门控」的根本原因。
2. **训练内化是主增益**。推理时注入只是「探路」，真正的价值在用训练把能力内化：重叠+噪声 cpWER 26–71 → 9–15，训练后不给线索也大幅变好，Context 变成了模型自身能力。
3. **越弱的模型收益越大**。这条规律在推理时、监督微调内化、强化学习内化三个层面一致成立：Ming 的降幅（−56 / −64.2）、迁移幅度（探针 +37pp）全程最大；连多目标张力问题也只出现在最小的 8B 上。强模型接近天花板、空间小；弱模型有大量失败可被线索救回。
4. **前置能力 + 上下文设计精准度决定成败**。弱前置 / 坏设计 / 坏措辞会反伤（−11.7 的灾难、注意力虹吸），强前置 + 好设计 + 好措辞翻盘（难样本 +1.3、流式 cpWER 全场最优）。线索质量不够时不是没用，是反伤——这条贯穿噪声场景的设计对比和流式场景的措辞实验。
5. **能力可内化、可迁移**。二元门控训练顺带提升了独立的听觉计数 / 场景判断推理（+10pp），且零灾难遗忘；强化学习调行为（160–240 条数据）、监督微调灌容量（3500 条数据），RL 的样本效率高一个量级，但依赖 SFT/预训练已有的表征。两者是分工而非替代。

贯穿这一切的底线是：**全程零泄漏**——静音探针 + 泄漏门禁守住，所有增益是真实理解而非抄答案。

# 十、诚实记录的负结果（界定适用边界）

一个可信的研究必须诚实记录它不工作的地方。本项目的负结果同时也界定了 AGSC 的适用边界：

- **干净 / 近天花板任务无增益空间**：S1 单人噪声转写（CER ~3%）、干净多说话人检测（98.8% 已近满分），模型本就强，没有可改善的空间。
- **强模型在简单样本上线索成噪声**：Qwen3 在 S2 简单样本本就会做，不完美线索轻微拖累，导致推理时全样本净增益微负（−1.8）。
- **前置不可靠则有害**：AST 声学场景标签 29% 准确率 → 弃用；带噪 pyannote 的光秃时间窗 → −1.6；ASR 整句错草稿 → −11.7。
- **流式超短前缀的物理边界**：1–2 秒前缀线索 0–2 词，无增益，弱模型甚至被空框架拖累；突破这个瓶颈需要前置组件低延迟化（流式分离 / 流式 ASR），而非 prompt 工程。
- **门控措辞模型相关**：gated_v2 的「仍需完整转写」对 Qwen3 是神来之笔，对 MiniCPM 却有副作用（话变啰嗦、插入错误增多）——没有万能 prompt。
- **多目标挤压在小模型上真实存在**：MiniCPM 门控 0.60 → 0.367，需 Eq.7/Eq.8 显式调节，修复有代价（内化回吐），是取舍不是免费午餐。
- **门控塌缩与 RL 的局限**：强化学习难以矫正已饱和的门控（饱和点零梯度），必须靠监督微调破饱和——这本身是一个有价值的方法学负结果。
- **SAM Audio 不适合同类人声分离**：通才不及专才（43–56% vs SepFormer 16.8%）。

# 十一、工程实现与避坑（开源价值）

本项目全程在一台 driver 470（只支持 CUDA ≤11.4）的老卡机器上单机完成，沉淀了一整套「在老硬件上跑通现代多模态训练」的避坑经验，这部分本身具有开源价值：

- **老驱动的 CUDA 墙贯穿全程**：所有 torch 必须用 cu118 版本；DeepSpeed 需要在环境内单独装 CUDA 11.8 的 nvcc 并设置 CUDA_HOME，否则它的 FusedAdam 会按系统的 CUDA 12.8 编译、生成 CUDA 12 kernel，在 470 上直接报「driver insufficient」。SAM Audio 的 torchcodec 依赖 CUDA 12 的 libnvrtc.so.13，是同源的硬墙。
- **依赖安全**：安装任何包一律加 `--no-deps`，防止某个依赖把 torch 偷偷升级到 cu13，破坏整个环境。
- **多环境隔离**：omni-context（Qwen3 + DeepSpeed）/ omni-context-mcpm（MiniCPM）/ ming / omni-pipeline（合成管线）/ samaudio-venv（SAM Audio）各自隔离，互不污染。
- **国内网络镜像**：conda 用 USTC、pip 用 Aliyun、github 用 ghfast.top、HuggingFace 用 hf-mirror.com。
- **三模型接口坑**：Ming-flash 攻克 7 个自定义接口坑（模板只接受 HUMAN 轮、PEFT task_type、whisper 音频 dtype、cache bool、fused-CE 的 inplace_backward 等）；MiniCPM 的 chat 采样参数名是 `do_sample=`（传 `sampling=` 会被静默吞掉——这是强化学习阶段一「采样失效」的真正根因，后来靠解析策略 + ε-探索绕过，阶段二修正参数名后启用真采样）。
- **强化学习工程要点**：RL 必须与部署用同一个模板（SFT 扛得住模板错位，RL 的微小推力扛不住）；组内零方差退化需要 ε-探索或温度采样兜底；并行评测的临时文件必须带 tag 防竞态。
- **门控塌缩攻坚的两个新坑**：用 `nohup bash -c "A; B"` 把命令串在后台轮询的 ssh 里，轮询 ssh 一收尾会把整串子进程杀掉（应各步独立 nohup）；`pkill -f "模式"` 如果模式串恰好出现在命令行自身会自杀（应按 pid kill）。

# 十二、局限与未来工作

1. **S1 受天花板限制**：需要更难的噪声集（更低 SNR、远场、非中文）才能看到 S1 上的增益空间。
2. **更难场景未覆盖**：3 人以上重叠 + 噪声、强混响、口音变化尚未系统验证。
3. **外部效度可扩**：WildSpeech-Bench / MMAU / VoiceBench（噪声子集）/ Dynamic-SUPERB Ph2 的 judge 评分尚未全量接入。
4. **训练规模**：当前主要是 1 epoch、较小 rank 的 LoRA；更长训练、更大 rank 值得探索。
5. **流式前置低延迟化**：1–2 秒超短前缀无增益的瓶颈在前置组件（SepFormer / ASR 需 ≥3 秒积累），流式分离与流式 ASR 可把增益窗口前移。
6. **置信感知线索**：根据前置组件的置信度自适应调节线索强度（管线已输出 high/medium/low 置信标注，接口已留）。
7. **门控塌缩的根本预防**：本项目是「先塌缩、再用 SFT 救」；未来可探索在训练分布上预防（提高 clean 占比、课程设计），从源头避免塌缩。

# 十三、结论、贡献与交付

## 命题验证

用「只给线索、不给答案、零泄漏、可自动生成」的 AGSC 上下文配合轻量训练内化，能让多个不同架构、不同规模的 Omni 模型，在多说话人重叠与环境噪声混合的复杂场景下，对 who-said-what 获得真实、可复现、抄不来的提升；再用一次强化学习把「门控 + 转写 + 流式线索利用」整条链路内化进模型；最后用监督微调破饱和，彻底矫正强化学习诱导的门控塌缩。命题成立，且每一步都经得起静音探针与难样本切片的检验。

## 主要贡献

1. **AGSC 零泄漏线索范式 + 静音探针验证**：一套判别「真懂 vs 抄答案」的方法学，三道保险（防抄契约 / 泄漏门禁 / 静音探针）可程序化、可复现。
2. **可泛化、可开源的合成流水线**：pyannote + Mega-ASR + SepFormer + VAD/SNR，每个组件都经实证选型（含 AST 标签 29% 弃用这样的实证否决）。
3. **强前置选型方法论**：用分离草稿 cpWER 实证对比，确立 SepFormer ≫ SAM Audio 用于同类人声分离。
4. **三种异构 Omni 模型的训练器**：DeepSpeed / 单卡 / naive-MP 三套 LoRA 训练器，含多音频接口适配。
5. **决定性实验结果**：重叠 + 噪声 cpWER 26–71 → 9–15，最难样本绝对改善 36–62pp，跨三模型一致、弱模型最大、全程非泄漏。
6. **实时性实测**：在线推理开销可忽略（prefill 文本增量 ≤155 token、墙钟 ≤0.21 秒），训练内化后零延迟部署。
7. **流式场景系统验证**：增量增益曲线（线索 3–6 秒成熟、6 秒达全量约 8 成）+ 实时门控（修复注意力虹吸、省 65% 注入量）+ 一 token 探针。
8. **GDPO 三阶段强化学习**：GDPO≻GRPO 受控对照、全链内化（无线索推理最高 −64.2）、模型自发门控反超外置检测器（0.53 → 0.75–0.80）、推理正迁移（+10pp）、多目标张力转化为可控帕累托。
9. **门控塌缩的诊断-矫正范式**：forced-choice π 探针定位概率层饱和 → 证伪软门控 → 解释 RL 失效 → SFT 破饱和 + GATE-with-transcript 双目标解决转写截断；方法学启示「诊断问题在哪一层、决定矫正手段」。
